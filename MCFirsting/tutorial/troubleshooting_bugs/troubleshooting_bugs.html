การแก้ไขปัญหาและบั๊กใน Add-Ons

คู่มือรับมือข้อผิดพลาดกวนใจ เพื่อให้ Add-ons ของคุณทำงานได้อย่างสมบูรณ์

ผู้สร้าง Add-ons ทุกคนล้วนต้องเผชิญกับ บั๊ก (Bugs) ครับ คู่มือฉบับสมบูรณ์นี้ได้รวบรวมปัญหาที่พบบ่อยที่สุด ที่คุณจะต้องเจอเมื่อสร้าง Add-ons สำหรับ Minecraft: Bedrock Edition

เนื้อหาในนี้จะอธิบายว่า ทำไมมันถึงเกิดขึ้น และแสดงวิธี แก้ไข (Fix) รวมถึงวิธี ป้องกัน (Prevent) ไม่ให้มันเกิดซ้ำขึ้นอีก

จงใช้บทความนี้เป็น แหล่งอ้างอิงหลัก ของคุณทุกครั้งที่มีบางอย่างในเกมทำงานไม่ปกติครับ!

รายการตรวจสอบเบื้องต้น

ก่อนที่จะเจาะลึกไปที่ปัญหาเฉพาะเจาะจง ลองตรวจสอบตามรายการเช็คลิสต์ด่วนนี้ดูก่อนครับ:

  • ตรวจสอบ Content Log: บั๊กส่วนใหญ่มักจะสร้างข้อความแจ้งเตือนข้อผิดพลาด (Error messages) ไว้ให้เราดูครับ
  • ตรวจสอบไวยากรณ์ (JSON Syntax): แนะนำให้ใช้โปรแกรมอย่าง VS Code ที่มีฟังก์ชันตรวจสอบ JSON เพื่อหาจุดผิดพิมพ์ตก
  • ยืนยันตำแหน่งไฟล์: ตรวจดูให้แน่ใจว่าวางไฟล์ถูกโฟลเดอร์ และตั้งชื่อไฟล์ถูกต้องครบถ้วน
  • จับคู่ Identifiers ให้ตรงเป๊ะ: ระบบจะแยกแยะตัวพิมพ์เล็ก/ใหญ่ ดังนั้นต้องพิมพ์ให้ตรงกันเป๊ะๆ และอย่าลืมใส่ Namespaces (เช่น demo:) ด้วยครับ
  • โหลดข้อมูลใหม่ให้ถูกต้อง: พิมพ์คำสั่ง /reload ในเกม หรือกดออกแล้วเข้าโลกใหม่ เพื่อให้เกมอัปเดตไฟล์ล่าสุด
  • ล้างแคชหากจำเป็น: บางครั้งข้อมูลเก่าๆ อาจยังตกค้างอยู่ ทำให้ไฟล์ใหม่ไม่ยอมทำงาน

การตั้งค่าสภาพแวดล้อมสำหรับการแก้บั๊ก

เปิดใช้งาน Content Log

Content Log ควรเป็นหนึ่งในเครื่องมือแก้บั๊กหลักที่คุณนึกถึงเสมอครับ คุณสามารถเปิดใช้งานมันได้ในหน้าต่างการตั้งค่า Creator ภายในเกม Minecraft ลองเข้าไปอ่านบทความเกี่ยวกับ Content Error Log เพื่อดูรายละเอียดแบบเต็มๆ เกี่ยวกับวิธีใช้งานได้เลยครับ!

*เมื่อเปิดใช้งานแล้ว หาก Add-ons ของคุณมีข้อผิดพลาด เกมจะมีหน้าต่างเด้งขึ้นมาบอกว่าไฟล์ไหนบรรทัดไหนที่มีปัญหา ช่วยลดเวลาคลำหาจุดผิดได้มหาศาลครับ

ตั้งค่า IDE ของคุณ

หากคุณใช้โปรแกรม Visual Studio Code (VS Code) เป็น IDE ในการเขียนโค้ด คุณควรติดตั้งส่วนเสริมเฉพาะสำหรับ Minecraft เอาไว้ด้วยครับ

ส่วนเสริมที่เราแนะนำให้ติดตั้งเพื่อช่วยจับผิดโครงสร้าง JSON คือ:

  • Minecraft Debugger
  • Blockception's Minecraft Bedrock Development

เครื่องมือเหล่านี้จะช่วยขีดเส้นใต้สีแดงเตือนคุณทันทีที่พิมพ์วงเล็บตก หรือใส่เครื่องหมายลูกน้ำ (Comma) ผิดที่ครับ!

คำสั่งที่เป็นประโยชน์สำหรับการทดสอบ

การใช้คำสั่งในเกมอย่างถูกต้องจะช่วยให้คุณจำลองสถานการณ์และหาจุดผิดพลาดได้ไวขึ้น นี่คือรายการคำสั่งที่สายทำ Add-ons ควรจำไว้ครับ:

  • /reload โหลด Behavior และ Resource packs ใหม่ (อัปเดตไฟล์โดยไม่ต้องออกเข้าโลกใหม่)
  • /give @s namespace:item_name ทดสอบเสก Items
  • /summon namespace:entity_name ทดสอบเสก Entity
  • /setblock ~ ~ ~ namespace:block ทดสอบวาง Blocks
  • /function my_function ทดสอบฟังก์ชัน
  • /script profiler start เริ่มเก็บข้อมูลประสิทธิภาพของ Script
  • /script profiler stop หยุดเก็บข้อมูลและดูผลลัพธ์

ข้อผิดพลาดด้านไวยากรณ์และ JSON (Syntax Errors)

ข้อผิดพลาดทางไวยากรณ์ของ JSON (Syntax Errors) เป็นปัญหาที่พบบ่อยที่สุด และมักจะเป็นปัญหาที่ แก้ไขได้ง่ายที่สุด เมื่อคุณรู้ว่าต้องโฟกัสที่จุดไหนครับ โปรแกรมอย่าง Visual Studio Code จะเน้น (Highlight) จุดที่ผิดพลาดเหล่านี้ให้คุณเห็นโดยอัตโนมัติ หากคุณเปิดใช้งานตัวตรวจสอบ JSON เอาไว้:

  • การลืมใส่หรือใส่ลูกน้ำเกิน:
    กฎของ JSON บังคับให้ต้องมีเครื่องหมายลูกน้ำ (Commas ,) คั่นระหว่างคุณสมบัติ ทั้งหมด ยกเว้นตัวสุดท้าย ที่อยู่ใน Object {} หรือ Array [] ครับ
  • วงเล็บไม่ครบคู่:
    เมื่อคุณเปิดวงเล็บปีกกา { หรือวงเล็บก้ามปู [ คุณจะต้องปิดมันด้วย } และ ] ในลำดับที่ถูกต้องเสมอ (เปิดตัวไหนหลังสุด ต้องปิดตัวนั้นก่อน)
  • ตัวอักษรที่ไม่ถูกต้อง:
    ระวังตัวอักษร Unicode ที่ซ่อนอยู่, ตัวอักษรที่ใช้ผิดรูปแบบ (เช่น เผลอก๊อปปี้ "เครื่องหมายคำพูดแบบโค้ง" มาจากโปรแกรมออฟฟิศอย่าง Word แทนที่จะใช้แบบตรง " "), หรือการแอบใส่คอมเมนต์ (JSON comments) ลงในไฟล์บางประเภทที่ไม่รองรับการคอมเมนต์ครับ

ปัญหาเกี่ยวกับชื่อและ Identifier

การตั้งชื่อและการอ้างอิงชื่อในไฟล์ต่างๆ ต้องมีความสม่ำเสมอและไม่ซ้ำซ้อนกับของเดิมครับ ลองตรวจสอบจุดเหล่านี้ดู:

  • UUID ต้องไม่ซ้ำกัน:
    คุณต้องตรวจสอบให้แน่ใจว่าแพ็กของคุณใช้ UUIDs ที่ไม่ซ้ำกัน (Unique UUIDs) ในไฟล์ manifest.json ไม่อย่างนั้นเกมอาจจะสับสนหรือโหลดแพ็กไม่ขึ้นครับ
  • Identifier ต้องตรงกันเป๊ะ:
    Identifier ที่ตั้งไว้ใน Resource packs และ Behavior packs จะต้องตรงกันทุกประการ (โปรดจำไว้ว่า Identifier นั้นแยกแยะตัวพิมพ์เล็ก-ใหญ่ หรือ Case-sensitive ด้วย: ดังนั้น custom_mob กับ Custom_Mob ถือว่าเป็นคนละตัวกันเลยนะครับ!)
  • ใช้ Namespaces เสมอ:
    การใช้ Namespaces (เช่น myaddon:custom_sword) จะช่วยหลีกเลี่ยงการทับซ้อนกับ Items ของเกมหลักหรือ Add-ons ตัวอื่นๆ ครับ
  • เลี่ยงชื่อสงวน:
    หลีกเลี่ยงการใช้ชื่อที่ถูกสงวนไว้สำหรับเนื้อหาต้นฉบับของเกม (Vanilla content) เพื่อไม่ให้ไปเขียนทับหรือทำให้ระบบของเกมรวนครับ

บั๊กเกี่ยวกับ Entity

ปัญหาเกี่ยวกับการสร้างและตั้งค่า Custom Entity ที่พบบ่อย มีดังนี้ครับ:

1. Entity ทะลุโลก

อาการ: Entity แบบกำหนดเองเกิดมาปุ๊บก็ร่วงหล่นลงไปใน Void ทันที

สาเหตุ: ขาด Physics Component

วิธีแก้: เพิ่มโค้ดต่อไปนี้ลงในส่วน "Components" ในไฟล์ Behavior ของ Entity

"Components": {
            "minecraft:physics": {
                "has_gravity": true,
                "has_collision": true
            }
        }

2. Entity ไม่ขยับ

อาการ: Entity เกิดมาได้แต่ยืนนิ่งสนิท ไม่ไปไหน

สาเหตุและวิธีแก้ที่พบบ่อย:

  • ขาด Components การเคลื่อนที่: 
    "Components": {
            "minecraft:movement": { "value": 0.25 },
            "minecraft:movement.basic": {},
            "minecraft:navigation.walk": {
                "can_path_over_water": true
            }
        }
  • ไม่มีเป้าหมาย AI: ให้เพิ่ม Components พฤติกรรม เช่น 
    "minecraft:behavior.random_stroll": {
            "priority": 6,
            "speed_multiplier": 1.0
        }
  • ความเร็วการเคลื่อนที่เป็นศูนย์: ตรวจสอบว่าค่า value ของ movement มีค่ามากกว่า 0

3. Entity ไม่มี Textures (ลายตารางสีชมพูดำ)

อาการ: Entity ปรากฏตัวพร้อมกับลวดลายตารางสีชมพูสลับดำ ซึ่งบ่งบอกว่า Textures หายไป

ขั้นตอนการตรวจสอบ:

  • ตรวจสอบ Texture Path ในไฟล์ Client Entity ว่าระบุตำแหน่งไฟล์ถูกต้องตรงกับของจริงหรือไม่
  • ตรวจสอบ Identifiers ของ Behavior และ Resource pack ว่าตรงกันเป๊ะ
  • ตรวจสอบให้แน่ใจว่าไฟล์ Textures เป็นไฟล์ PNG ที่ถูกต้อง (ไม่ใช่แค่เอาไฟล์อื่นมารีเนมเปลี่ยนนามสกุล)
  • ตรวจสอบ Render Controller ว่าได้อ้างอิงชื่อ Textures ถูกต้องหรือไม่

4. Entity ไม่เกิดเองตามธรรมชาติ

อาการ: Entity ปรากฏตัวได้แค่จากการใช้ไข่เกิด (Spawn egg) หรือคำสั่งเท่านั้น

สาเหตุ: ขาดไฟล์ หรือตั้งค่ากฎการเกิด (Spawn Rules) ผิดพลาด

วิธีแก้: สร้างไฟล์ Spawn Rule ขึ้นมา (ในโฟลเดอร์ `spawn_rules`) ตัวอย่างเช่น: 

{
            "format_version": "1.8.0",
            "minecraft:spawn_rules": {
                "description": {
                    "identifier": "mypack:my_entity",
                    "population_control": "animal"
                },
                "conditions": [
                    {
                        "minecraft:spawns_on_surface": {},
                        "minecraft:brightness_filter": {
                            "min": 7,
                            "max": 15
                        },
                        "minecraft:weight": {
                            "default": 50
                        }
                    }
                ]
            }
        }

5. Components Groups ไม่ทำงาน

อาการ: ฟีเจอร์ต่างๆ ที่ตั้งไว้ใน Component Groups ไม่ยอมแสดงผล

ปัญหา: Component Groups จำเป็นต้องมี เหตุการณ์ (Events) เป็นตัวกระตุ้นให้มันทำงานครับ

"component_groups": {
            "angry_mode": {
                "minecraft:attack": { "damage": 10 }
            }
        },
        "events": {
            "become_angry": {
                "add": {
                    "component_groups": ["angry_mode"]
                }
            }
        }

*คุณต้องกระตุ้น Trigger Event นี้ผ่านเงื่อนไขบางอย่าง เช่น เหตุการณ์ตอนเกิด (Spawn event), การตรวจจับ (Sensor), หรือการโต้ตอบ (Interaction) ด้วยครับ

6. Animation ไม่เล่น

อาการ: Entity แข็งทื่อ ไม่มี Animation หรือเล่น Animation ผิดท่า

รายการตรวจสอบ:

  • ตรวจสอบว่ามี Animation controller อยู่จริง และถูกอ้างอิงไว้ในไฟล์ Client Entity แล้ว
  • ตรวจสอบว่าไฟล์ Animation มี Identifiers ที่ถูกต้องตรงกัน
  • การเปลี่ยนสถานะ (State transitions) ใน Controller ใช้ Query Conditions ที่ถูกต้อง
  • มีการอ้างอิง Animation ด้วย ชื่อย่อที่ถูกต้อง ในส่วนของ "animations" หรือ "scripts"

บั๊กเกี่ยวกับ Blocks

ปัญหาเกี่ยวกับการสร้างCustom Blocksง (Custom Blocks) ที่พบได้บ่อย มีดังนี้ครับ:

1. Blocks ไม่มี Texture

อาการ: Blocks แสดงผลเป็นสีทึบๆ สีเดียว หรือลวดลาย Texture หายไป

รายการตรวจสอบ:

  • ตรวจสอบไฟล์ terrain_texture.json ว่ามีรายการ (Entry) ระบุถึง Texturesของคุณแล้ว
  • ชื่อ Texture ในไฟล์ Block JSON ต้อง ตรงกัน กับชื่อที่ตั้งไว้ใน terrain_texture
  • ไฟล์ภาพ Texture มีอยู่จริงตาม (Path) ที่ระบุเอาไว้
  • ขนาดของ Texture ต้องเป็น 16x16, 32x32, 64x64 เป็นต้น

2. Blocks ไม่ดรอป Items

อาการ: เมื่อทุบหรือขุด Custom Blocks แล้ว ไม่มีอะไรดรอปออกมาเลย

วิธีแก้: ตรวจสอบให้แน่ใจว่าคุณได้อ้างอิงตารางของดรอป (Loot table) และไฟล์ Loot table นั้นมีอยู่จริงในแพ็กครับ

ในไฟล์ Block JSON ส่วน "components":

"components": {
          "minecraft:loot": "loot_tables/blocks/my_block.json"
        }

ตัวอย่างตาราง Loot table แบบง่าย (loot_tables/blocks/my_block.json):

{
          "pools": [
            {
              "rolls": 1,
              "entries": [
                {
                  "type": "item",
                  "name": "mypack:my_block"
                }
              ]
            }
          ]
        }

3. Custom Geometry ไม่แสดง

อาการ: Blocks แสดงผลเป็นลูกบาศก์เริ่มต้น (Default cube) แทนที่จะเป็นโมเดลแบบกำหนดเอง (Custom model) ที่เราตั้งไว้

รายการตรวจสอบ:

  • ไฟล์ Geometry ถูกเก็บไว้ในโฟลเดอร์ models/blocks/ ถูกต้องหรือไม่
  • Identifier ของ Geometry ตรงกันกับที่อ้างอิงไว้ในไฟล์ Block JSON หรือไม่
  • ไฟล์ Block JSON ใช้ไวยากรณ์อ้างอิง (Syntax) ที่ถูกต้อง ดังนี้:
"minecraft:geometry": {
          "identifier": "geometry.my_custom_block"
        }

4. สถานะ Blocks ไม่ยอมเปลี่ยน

อาการ: การสลับรูปแบบ Blocks (Block permutations) ไม่ทำงานอย่างที่ควรจะเป็น

สาเหตุที่พบบ่อย:

  • ไม่ได้กำหนดค่าสถานะ (State values) เอาไว้ในส่วน description
  • เขียนไวยากรณ์คำสั่งเงื่อนไข (Condition query syntax) ผิดพลาด
  • ไม่ได้ใช้เหตุการณ์ (Event) เพื่อสั่งตั้งค่าสถานะอย่างถูกต้อง
{
          "description": {
            "identifier": "mypack:my_block",
            "states": {
              "mypack:active": [false, true]
            }
          },
          "permutations": [
            {
              "condition": "query.block_state('mypack:active') == true",
              "components": { ... }
            }
          ]
        }

บั๊กเกี่ยวกับ Items

ปัญหาเกี่ยวกับการสร้างและตั้งค่า Items แบบกำหนดเอง (Custom Items) ที่พบบ่อย มีดังนี้ครับ:

1. Items ไม่อยู่ในช่อง Creative inventory

อาการ: ไม่สามารถหา Items เจอในหน้าต่าง Creative inventory ทำได้แค่เสกผ่านคำสั่งเท่านั้น

วิธีแก้: เพิ่มหมวดหมู่เมนู (Menu category) เข้าไปในไฟล์ Item JSON ของคุณ

"description": {
            "identifier": "mypack:custom_item",
            "menu_category": {
                "category": "equipment",
                "group": "itemGroup.name.sword"
            }
        }

*หมวดหมู่ที่ใช้ได้ ได้แก่: construction, nature, equipment, items, และ none

2. Items แสดง Textures ผิด

อาการ: Items แสดงผลเป็น Textures เริ่มต้น หรือใช้ Textures ผิดเพี้ยนไปจากที่ตั้งไว้

รายการตรวจสอบ:

  • ตรวจสอบว่ามีไฟล์ item_texture.json อยู่ในโฟลเดอร์ textures แล้ว
  • ชื่อ Texturesที่อ้างอิงในไฟล์ Item JSON ต้องตรงกันเป๊ะกับรายการ (Entry) ที่อยู่ใน item_texture
  • ไฟล์ภาพ Texturesมีอยู่จริงตามพาธที่ระบุเอาไว้

ตัวอย่างไฟล์ item_texture.json:

{
            "resource_pack_name": "My Pack",
            "texture_name": "atlas.items",
            "texture_data": {
                "my_item": {
                    "textures": "textures/items/my_item"
                }
            }
        }

3. เครื่องมือขุดแร่ทำงานไม่ถูกต้อง

อาการ: เครื่องมือแบบกำหนดเอง (เช่น Shovel, Pickaxe) ไม่สามารถขุด Blocks ที่ควรจะขุดได้ หรือขุดได้ช้าเกินไป

วิธีแก้: ตั้งค่า Digger Component ให้ถูกต้องตามเป้าหมายของเครื่องมือ 

"minecraft:digger": {
          "use_efficiency": true,
          "destroy_speeds": [
            {
              "block": {
                "tags": "query.any_tag('stone', 'metal')"
              },
              "speed": 8
            }
          ]
        }

บั๊กเกี่ยวกับ Script

ปัญหาเกี่ยวกับการเขียน Scripting API ด้วย JavaScript ที่พบบ่อย มีดังนี้ครับ:

1. Script ไม่โหลด

อาการ: ฟังก์ชันของ Script ไม่ทำงานเลย และไม่มีแจ้งเตือนข้อผิดพลาดใดๆ ใน Log

รายการตรวจสอบ:

  • ตรวจสอบในไฟล์ manifest.json ว่ามี Module Script และกำหนด "type": "script" เอาไว้แล้ว
  • Entry Path ชี้ไปที่ไฟล์ถูกต้อง เช่น "entry": "scripts/main.js"
  • ไฟล์ Script มี Syntax JavaScript ที่ถูกต้อง
  • ส่วน Dependencies ต้องใช้ module_name เท่านั้น (ห้ามใช้ uuid):
"dependencies": [
            {
                "module_name": "@minecraft/server",
                "version": "1.17.0"
            }
        ]

2. ข้อผิดพลาด "Cannot read property of undefined"

อาการ: Script Crash และแสดงข้อความ Error นี้ขึ้นมา

ปัญหา: คุณกำลังพยายามเข้าถึงค่าหรือคุณสมบัติ (Properties) จากตัวแปรที่ว่างเปล่า หรือค่าที่ไม่มีอยู่จริง (Undefined values)

// โค้ดนี้จะแครชทันทีถ้าไม่มีผู้เล่นออนไลน์อยู่เลย
        const player = world.getAllPlayers()[0];
        const x = player.location.x;  // Error เกิดตรงนี้เพราะ player คือ undefined

วิธีแก้: ตรวจสอบความถูกต้อง (Validate) เสมอก่อนที่จะดึงค่าไปใช้:

const players = world.getAllPlayers();
        if (players.length > 0) {
          const player = players[0];
          const x = player.location.x;
        }

3. เหตุการณ์ทำงานซ้ำซ้อนหลังการ Reload

อาการ: เหตุการณ์ (Events) ทำงานหรือถูกกระตุ้นหลายครั้งผิดปกติหลังจากพิมพ์คำสั่ง /reload

ปัญหา: การโหลดซ้ำแต่ละครั้ง จะเป็นการเพิ่มโค้ดการสมัครรับข้อมูล (Subscription) ซ้อนทับเข้าไปเรื่อยๆ

วิธีแก้: ใช้ตัวแปรเช็คสถานะการเริ่มต้น (Initialize check) เพื่อป้องกันการสมัครรับ Event ซ้ำ:

let isInitialized = false;
        
        function initialize() {
          if (isInitialized) return;
          
          world.afterEvents.playerSpawn.subscribe(onPlayerSpawn);
          isInitialized = true;
        }
        
        initialize();

4. Script รันผ่าน แต่ไม่มีอะไรเกิดขึ้น

อาการ: Script ไม่มีแจ้งเตือน Error ใดๆ แต่พฤติกรรมที่เราคาดหวังก็ดันไม่ทำงานเหมือนกัน

ขั้นตอนการแก้ปัญหา (Debugging Steps):

  • เพิ่มคำสั่ง console.log ลงไปตามจุดต่างๆ เพื่อตรวจดูว่าโค้ดบรรทัดนั้นถูกรันจริงหรือไม่ (Verify code execution)
  • ตรวจสอบว่าคุณกำลังใช้ประเภทเหตุการณ์ (Event) ถูกต้องหรือไม่ (เช่น การสับสนระหว่าง beforeEvents กับ afterEvents)
  • ตรวจสอบชื่อของ Entity/Block ว่าตรงกันเป๊ะรวมถึง Namespace ด้วย
  • ตรวจสอบให้แน่ใจว่าคุณไม่ได้เผลอไปสั่งยกเลิกเหตุการณ์ (Canceling events) โดยไม่ได้ตั้งใจ
  • คุณสามารถเข้าไปอ่านเทคนิคการแก้ปัญหาแบบเจาะลึกได้ในบทความอ้างอิงเรื่อง Debugging Scripts ครับ

บั๊กเกี่ยวกับสูตรคราฟต์ (Recipe)

ปัญหาเกี่ยวกับการสร้างสูตรคราฟต์ (Crafting recipes) หรือการหลอม Items ที่พบบ่อย มีดังนี้ครับ:

1. สูตรคราฟต์ไม่ทำงาน

อาการ: ใส่ Items วัตถุดิบลงไปแล้วแต่ไม่ได้ผลลัพธ์ (Output) ออกมา

รายการตรวจสอบ:

  • ตรวจสอบว่าไฟล์สูตรคราฟต์ถูกเก็บไว้ในโฟลเดอร์ recipes/ แล้วหรือยัง
  • ประเภทของสูตร (Recipe type) ตรงกับวิธีการคราฟต์ที่ต้องการ (เช่น recipe_shaped, recipe_shapeless)
  • Identifiers ของ Items มีการใส่ Namespace ครบถ้วน (เช่น mypack:)
  • รูปแบบการวาง (Pattern) และคีย์ (Keys) ถูกต้องและสอดคล้องกัน

ตัวอย่างสูตรคราฟต์แบบมีรูปทรงที่ถูกต้อง: 

{
          "format_version": "1.21.40",
          "minecraft:recipe_shaped": {
            "description": {
              "identifier": "mypack:custom_recipe"
            },
            "tags": ["crafting_table"],
            "pattern": [
              "###",
              " | ",
              " | "
            ],
            "key": {
              "#": { "item": "minecraft:diamond" },
              "|": { "item": "minecraft:stick" }
            },
            "unlock": [
              { "item": "minecraft:diamond" }
            ],
            "result": {
              "item": "mypack:custom_sword"
            }
          }
        }

2. สูตร Furnace ไม่ทำงาน

อาการ: Items ไม่ยอมเผาหรือหลอม (Smelt) เมื่อใส่ลงในเตา

วิธีแก้: ตรวจสอบว่าคุณใช้ Tags ที่ถูกต้องให้ตรงกับประเภทของเตาเผาครับ 

{
          "format_version": "1.21.40",
          "minecraft:recipe_furnace": {
            "description": {
              "identifier": "mypack:smelt_custom_ore"
            },
            "tags": ["furnace", "blast_furnace"],
            "input": "mypack:custom_ore",
            "output": "mypack:custom_ingot"
          }
        }

*Tags ที่ใช้บ่อย: furnace (เตาเผาปกติ), blast_furnace (เตาหลอมแร่), smoker (เตารมควันอาหาร), campfire (แคมป์ไฟ)

บั๊กเกี่ยวกับ Loot table

ปัญหาเกี่ยวกับการดรอป Items จาก Entity หรือ Blocks ที่พบบ่อย มีดังนี้ครับ:

1. ของไม่ดรอป

อาการ: เมื่อฆ่า Entity หรือทุบ Blocks แล้วไม่มีอะไรดรอปออกมาเลย

รายการตรวจสอบ:

  • ตรวจสอบว่ามีไฟล์ Loot table อยู่ในโฟลเดอร์ loot_tables/ แล้วหรือยัง
  • Entity หรือ Blocks (Entity/block) อ้างอิง Path ของ Loot table ถูกต้อง
  • ไวยากรณ์ (Syntax) ในไฟล์ Loot table ถูกต้องครบถ้วน

ตัวอย่างตาราง Loot table พื้นฐาน:

{
          "pools": [
            {
              "rolls": 1,
              "entries": [
                {
                  "type": "item",
                  "name": "mypack:custom_item",
                  "weight": 1
                }
              ]
            }
          ]
        }

2. ของดรอปตามเงื่อนไขไม่ทำงาน

อาการ: เงื่อนไข (Conditions) ที่ตั้งไว้ไม่มีผลต่อการดรอปของอย่างที่คาดหวังไว้

วิธีแก้: ตรวจสอบไวยากรณ์ของเงื่อนไข (Condition syntax) ให้ถูกต้อง (สังเกตว่า conditions จะอยู่ในระดับเดียวกับ entries หรือภายในแต่ละ entry ก็ได้ ขึ้นอยู่กับเป้าหมาย): 

{
          "pools": [
            {
              "rolls": 1,
              "entries": [
                {
                  "type": "item",
                  "name": "mypack:rare_drop",
                  "weight": 1
                }
              ],
              "conditions": [
                {
                  "condition": "random_chance_with_looting",
                  "chance": 0.1,
                  "looting_multiplier": 0.05
                }
              ]
            }
          ]
        }

บั๊กด้านประสิทธิภาพ (Performance)

ปัญหาเกี่ยวกับความลื่นไหล การกินทรัพยากรเครื่อง และความเสถียรของเกมที่พบบ่อย มีดังนี้ครับ:

1. โลกโหลดช้า

สาเหตุที่เป็นไปได้:

  • มี Textures เยอะเกินไป หรือไฟล์ภาพมีขนาดใหญ่เกินไป
  • มีการเกิดของ Entity มากเกินไปในขณะโหลดโลก
  • มี Scripts ที่ซับซ้อน ถูกสั่งให้ทำงานทันทีตอนเริ่มต้นโลก

วิธีแก้ปัญหา:

  • ปรับลดขนาดและปรับแต่ง Textures (แนะนำให้ใช้ 16x16 หรือ 32x32 เมื่อเป็นไปได้)
  • เลื่อนการเริ่มต้นทำงานของ Scripts ที่ไม่จำเป็นออกไปก่อน
  • ลดขีดจำกัดความหนาแน่นของ Entity ในกฎการเกิด

2. เฟรมเรตตกขณะเล่นเกม

สาเหตุที่พบบ่อย:

  • มี Entity กระจุกตัวอยู่เยอะเกินไปในพื้นที่เดียว
  • มี Scripts ที่ถูกสั่งให้ทำงานทุกๆ tick ตลอดเวลา
  • Particle effects มีความซับซ้อนสูงเกินไป
  • มีการอัปเดต Blocks แบบไดนามิกมากเกินไป

วิธีแก้ปัญหา:

// แทนที่จะให้รันทุกๆ Tick ให้ใช้ Interval แทน
        system.runInterval(() => {
          // ใส่ Logic ของคุณที่นี่
        }, 20);  // ตั้งค่า 20 Ticks = รันแค่ 1 ครั้งต่อวินาที
        
        // จำกัดขอบเขตการประมวลผล
        const nearbyEntities = dimension.getEntities({
          location: player.location,
          maxDistance: 50  // ค้นหาในระยะ 50  Blocks  ไม่ต้องประมวลผลทั้งโลก
        });

3. ปัญหาหน่วยความจำของ Scripts

อาการ: เกมลื่นไหลดีในตอนแรก แต่จะค่อยๆ ช้าลง กระตุก หรือค้างเมื่อเวลาผ่านไปสักพัก

สาเหตุที่พบบ่อย:

  • การเก็บข้อมูลลงใน Arrays หรือ Maps ที่ใหญ่ขึ้นเรื่อยๆ โดยไม่มีการเคลียร์ค่าทิ้ง
  • การสมัครรับเหตุการณ์ (Event subscriptions) ค้างไว้และไม่ยอมลบออก (Remove) เมื่อไม่ใช้งานแล้ว
  • การแคช (Caching) ข้อมูลไว้ในหน่วยความจำมากเกินไป

วิธีแก้:

// เคลียร์ข้อมูลทิ้งเมื่อผู้เล่นออกจากเกม
        world.afterEvents.playerLeave.subscribe((event) => {
          playerDataMap.delete(event.playerId);
        });
        
        // จำกัดขนาดของแคช
        if (cache.size > 1000) {
          const firstKey = cache.keys().next().value;
          cache.delete(firstKey);
        }

*สามารถศึกษาข้อมูลเพิ่มเติมได้ที่บทความ: การปรับปรุงประสิทธิภาพและการใช้ทรัพยากร
