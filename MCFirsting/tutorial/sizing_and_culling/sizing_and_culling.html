ขนาดและการตัดส่วนที่มองไม่เห็น Sizing & Culling

สร้างบล็อกขนาดยักษ์ และเรียนรู้เทคนิคการซ่อนผิวเพื่อประสิทธิภาพสูงสุด

Custom Blocks ไม่ได้ถูกจำกัดอยู่แค่ขนาดสี่เหลี่ยมมาตรฐาน 16x16x16 ของ Minecraft อีกต่อไปครับ! นอกจากคุณจะสามารถสร้างบล็อกที่ ใหญ่กว่า 16x16x16 พิกเซลได้แล้ว ตั้งแต่เวอร์ชัน 1.20.60 เป็นต้นมา คุณยังสามารถใช้ฟีเจอร์ที่เรียกว่า "Culling" (การคัดออก) เพื่อลบชิ้นส่วนของบล็อกออกโดยอัตโนมัติ เมื่อมันถูกวางติดกับบล็อกอื่นที่บังด้านนั้นๆ อยู่

ในบทเรียนนี้ ขั้นแรกเราจะมาดูวิธีสร้าง บล็อกขนาดยักษ์ (Over-sized block) กันก่อน จากนั้นเราจะนำตัวอย่างจากบทเรียนก่อนหน้า (บล็อกซูชิ) มาสาธิตเรื่อง Culling ให้ดูครับ

สิ่งที่ต้องเตรียมตัวก่อนเริ่

ขอแนะนำให้คุณเรียนรู้และทำบทเรียนเหล่านี้ให้เสร็จเรียบร้อยก่อนนะครับ:

  • การสร้าง Custom Block
  • การสร้างบล็อกขั้นสูง (Advanced Custom Blocks)
  • การแสดงผลและแสงเงา (Render and Lighting Options)

การตั้งค่า Resource Pack

สร้างรูปทรงใน Blockbench

เรามาลองสร้างบล็อกขนาด 30x30x30 พิกเซล ใน Blockbench แต่ก่อนอื่น เราต้องมาทำความเข้าใจเกี่ยวกับข้อจำกัดของการสร้าง Custom Block ขนาดใหญ่กันเสียก่อนครับ

ข้อจำกัดของ Custom Block

ก่อนลงมือปั้นโมเดล เราต้องทราบข้อจำกัด 3 ข้อของการสร้างบล็อกขนาดใหญ่กันก่อนครับ:

  1. ขนาดจำกัด: บล็อกของคุณมีขนาดใหญ่ได้สูงสุดที่ 30x30x30 พิกเซล
  2. ขอบเขต: ขอบเขตสูงสุดของตำแหน่งบล็อกคือ 30 พิกเซลในทุกทิศทาง จากจุดกำเนิด (Origin)
    (จุดกำเนิดคือจุดกึ่งกลางที่ฐานด้านล่างสุดของบล็อกมาตรฐาน 16x16x16)
    นั่นหมายความว่าคุณมีพื้นที่ 30 พิกเซลไปทาง +x, -x, +y, -y, +z, และ -z ครับ บล็อกของคุณจะวางตรงไหนก็ได้ในเขตนี้นี้ ตราบใดที่ทำตามกฎข้อที่ 3
  3. ต้องแตะฐาน (Base Constraint): ข้อนี้สำคัญมาก! อย่างน้อย 1 พิกเซลของโมเดลคุณในแต่ละแกน ต้องซ้อนทับหรืออยู่ภายใน พื้นที่ของบล็อกฐาน (16x16x16) เสมอ

นี่คือภาพจำลองขอบเขตสูงสุด (Absolute bounds) ที่บล็อกของคุณต้องอยู่ภายใน เมื่อเทียบกับบล็อกฐาน 16x16x16 ครับ:

ตัวอย่างที่ถูกต้อง (VALID)

ด้านล่างนี้คือตัวอย่างของ Custom Block ขนาด 30x30x30 ที่ ถูกต้อง คืออยู่ภายในขอบเขต และมีส่วนของลูกบาศก์ (Cube) อย่างน้อยหนึ่งชิ้นทับซ้อนกับบล็อกฐาน:

ตัวอย่างที่ไม่ถูกต้อง (INVALID)

ส่วนนี่คือตัวอย่างที่ ผิด (Invalid) แม้ว่าจะอยู่ในขอบเขต 30x30x30 ก็ตาม แต่เป็นเพราะว่า ไม่มีส่วนใดสัมผัสหรือทับซ้อนกับบล็อกฐาน 16x16x16 เลย จึงไม่สามารถใช้งานได้ครับ:

สร้างส่วนที่เหลือของร่มยักษ์

สุดท้ายนี้ คุณจะต้องลุยเดี่ยวในการตั้งค่าส่วนที่เหลือของบล็อกร่มยักษ์ด้วยตัวเองแล้วครับ จำไว้นะครับว่า คุณทำได้แน่นอน!

นี่คือขั้นตอนที่คุณได้ทำมาแล้วทุกครั้งในการสร้าง Custom Block ลองทบทวนดูนะครับ:

Behavior Pack

ให้สร้างไฟล์ umbrella.json ขึ้นมา (ในโฟลเดอร์ blocks ของ Behavior Pack) และกรอกข้อมูลลงไปโดยใช้ความรู้ที่คุณได้เรียนมาจากบทเรียนก่อนหน้านี้ครับ

ติดปัญหาหรือเปล่า? (Troubleshooting Tips)

ถ้าคุณรู้สึกงงหรือทำไม่ได้ ไม่ต้องตกใจครับ เป็นเรื่องปกติมาก ลองเช็คตามรายการนี้ดู:

  • ย้อนกลับไปดูบล็อกซูชิ: ขั้นตอนการเขียนโค้ดเหมือนกันเป๊ะเลยครับ! (แค่เปลี่ยนชื่อ)
  • ตรวจสอบไฟล์เก่าๆ: ลืมสร้างไฟล์ไหนไปหรือเปล่า?
  • เช็คการก๊อปปี้: ก๊อปปี้ไฟล์มาแล้วลืมแก้ชื่อข้างใน (Identifier) ให้เป็น umbrella หรือเปล่า?
  • เทียบกับตัวอย่าง: ลองตรวจสอบงานของคุณกับไฟล์ตัวอย่างใน minecraft-samples ดูครับ
คลิกเพื่อดูเฉลยโค้ด umbrella.json

นี่คือหน้าตาของโค้ดที่ถูกต้องครับ:

{
          "format_version": "1.21.80",
          "minecraft:block": {
            "description": {
              "identifier": "demo:umbrella"
            },
            "components": {
              "minecraft:geometry": "geometry.umbrella",
              "minecraft:material_instances": {
                "*": {
                  "texture": "umbrella"
                }
              }
            }
          }
        }

การทดสอบบล็อก

เมื่อคุณกำหนดค่าบล็อกร่มยักษ์ทั้งใน Behavior Pack และ Resource Pack เสร็จเรียบร้อยแล้ว ก็ถึงเวลาทดสอบในเกมครับ

  1. เปิดช่องแชทขึ้นมา
  2. พิมพ์คำสั่งต่อไปนี้:
/give @s demo:umbrella

ทีนี้ก็เริ่มวางร่มแสนสวยพวกนั้นได้เลยครับ!

Note

สำหรับบล็อกที่มีขนาดใหญ่กว่าลูกบาศก์ฐาน 16x16x16 พิกเซล ส่วนของบล็อกที่ยื่นออกมานอกเขต 16x16x16 นั้นจะ ซ้อนทับ (Overlap) กับบล็อกอื่นๆ ที่อยู่ข้างเคียง

โปรดระวังเรื่องนี้ด้วยนะครับเมื่อคุณสร้างบล็อกขนาดใหญ่ เพื่อให้แน่ใจว่าคุณจะได้ภาพลักษณ์ที่ต้องการ โดยเฉพาะอย่างยิ่งถ้าบล็อกยักษ์ของคุณจะต้องวางอยู่ใกล้ชิดกับบล็อกอื่นๆ ครับ

การตัดส่วนที่มองไม่เห็น Culling

คุณสามารถทำให้ Custom Block ของคุณทำงานเหมือนกับบล็อกปกติในเกม (Vanilla blocks) เวลาที่วางติดกันหลายๆ บล็อก โดยการลบ "หน้าผิวที่มองไม่เห็นและซ้อนทับกัน" ออกด้วยเทคนิคที่เรียกว่า "Culling"

การทำ Culling จะช่วย เพิ่มประสิทธิภาพ (ลดอาการแลค) และ ลดโอกาสเกิดภาพบั๊ก (Graphical glitches) ได้อย่างมากครับ

เราจะมาลองทำสิ่งนี้กับ "Tuna Roll" (บล็อกซูชิที่เราเคยทำไปแล้ว) กันครับ โดยต้องเพิ่มไฟล์ทั้งใน Behavior Pack และ Resource Pack

1. Behavior Pack

ให้เข้าไปที่โฟลเดอร์ blocks และเปิดไฟล์ tuna_roll.json ขึ้นมา

เราจะแก้ไขส่วน minecraft:geometry จากเดิมที่เป็นแค่ชื่อ (String) ให้กลายเป็น วัตถุ (Object) ที่ประกอบด้วย identifier และ culling ดังนี้: 

"components": {
          "minecraft:geometry": {
            "identifier": "geometry.sushi",
            "culling": "test:sushi_cull"
          },
        }

นี่คือหน้าตาของไฟล์ tuna_roll.json แบบเต็มๆ หลังจากแก้ไขแล้วครับ:

{
          "format_version": "1.21.80",
          "minecraft:block": {
            "description": {
              "identifier": "demo:tuna_roll"
            },
            "components": {
              "minecraft:geometry": {
                "identifier": "geometry.sushi",
                "culling": "test:sushi_cull"
              },
              "minecraft:material_instances": {
                "north": "sushi_side",
                "south": "sushi_side",
                "*": {
                  "texture": "sushi_wrap"
                },
                "sushi_side": {
                  "texture": "tuna_roll"
                }
              }
            }
          }
        }

2. Resource Pack

ที่หน้าหลักของ Resource Pack ให้สร้างโฟลเดอร์ใหม่ชื่อว่า block_culling
