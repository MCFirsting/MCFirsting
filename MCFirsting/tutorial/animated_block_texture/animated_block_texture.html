การสร้าง Texture เคลื่อนไหว Animated Block Texture

เปลี่ยนบล็อกนิ่งๆ ให้ขยับได้ด้วยเทคนิค Flipbook Animation

ในบทเรียนนี้ คุณจะได้เรียนรู้วิธีสร้างอนิเมชันให้กับ Texture ของบล็อกโดยใช้ Resource Pack และข่าวดีคือ คุณไม่จำเป็นต้องใช้โปรแกรมทำอนิเมชันภายนอกเลยครับ!

หลักการทำงาน (The Flipbook Style)

เราจะใช้สไตล์ที่เรียกว่า "Flipbook animation" (อนิเมชันสมุดพลิก) ครับ เคยเห็น (หรือเคยทำ) ภาพวาดการ์ตูนตรงมุมสมุดหนังสือเรียนไหมครับ? ที่พอเรากรีดกระดาษเร็วๆ ภาพวาดเหล่านั้นจะดูเหมือนขยับได้? นั่นแหละครับคือสิ่งที่เรากำลังจะทำ

ภาพวาดแต่ละภาพคือ "เฟรม" (Frame) ของอนิเมชัน ภาพลวงตาของการเคลื่อนไหวจะเกิดขึ้นจากความเร็วในการเปลี่ยนเฟรม และความเนียนของการเชื่อมต่อระหว่างเฟรมเหล่านั้น

การจัดเก็บไฟล์

ในระบบ Flipbook animation ของ Minecraft เราจะไม่แยกไฟล์รูปภาพสำหรับแต่ละเฟรมออกจากกันครับ แต่จะมีเพียง ไฟล์เดียว และเฟรมทั้งหมดจะถูกเรียงต่อกันเป็น "แถบยาวๆ" (Long strip) (หลักการเดียวกับไฟล์ Animated GIF เลยครับ)

แต่ละเฟรมคือสี่เหลี่ยมขนาด 16x16 พิกเซล โค้ดอนิเมชันภายในเกมจะกำหนดหมายเลขให้แต่ละสี่เหลี่ยม โดยเริ่มนับจากด้านบนสุดเป็น เฟรมที่ 0

โดยปกติแล้ว บล็อกทั้ง 6 ด้านจะแสดงเฟรมเดียวกันพร้อมกัน แต่เดี๋ยวเราจะมาคุยกันเรื่องการใส่อนิเมชันที่แตกต่างกันในแต่ละด้านทีหลังครับ

สิ่งที่ต้องเตรียมตัวก่อนเริ่ม

ก่อนเริ่มบทเรียนนี้ คุณควรทำสิ่งเหล่านี้ให้เรียบร้อยครับ:

  • สร้าง Resource Pack พื้นฐาน (ดูได้ที่บทเรียน การสร้าง Resource Pack)
  • ดาวน์โหลด sample pack collection เก็บไว้ในเครื่องเพื่อดูตัวอย่าง
  • ดาวน์โหลด Vanilla Minecraft packs ตัวล่าสุดเพื่อใช้เป็นแหล่งอ้างอิงโค้ดและกราฟิก

โครงสร้าง Resource Pack สำหรับอนิเมชัน

นี่คือโครงสร้างพื้นฐานและเนื้อหาภายในสำหรับ Animation Pack ที่เรากำลังจะสร้างกันครับ:

        resource_pack/
            textures/
                blocks/
                    block_name.png
                flipbook_texture.json
            manifest.json

ลองเปิด Resource Pack จากตัวอย่าง Vanilla Minecraft sample packs ดูนะครับ

  1. เข้าไปที่โฟลเดอร์ textures แล้วตามด้วย blocks ตรงนี้แหละครับคือที่อยู่ของไฟล์ภาพ .png ของบล็อกต่างๆ
  2. เปิดไฟล์ flipbook_texture.json ขึ้นมาดูเนื้อหาข้างใน คุณจะเห็นรายชื่อ Texture แบบ Flipbook จำนวนมาก ลองสังเกตดูนะครับว่า Texture แต่ละอันมีการตั้งค่าที่แตกต่างกันออกไป

ตัวอย่างการตั้งค่า

1. Sculk (สคัลค์)

ตัวอย่างแบบง่าย มีการกำหนดความเร็วในการเปลี่ยนเฟรม

{
            "flipbook_texture": "textures/blocks/sculk",
            "atlas_tile": "sculk",
            "ticks_per_frame": 20
        }

2. Bubble Column (เสาฟองอากาศ)

ตัวอย่างที่ซับซ้อนขึ้น มีการเรียงลำดับเฟรมแบบย้อนหลัง (จาก 31 ไป 0)

{
            "flipbook_texture": "textures/blocks/bubble_column_inner_b",
            "atlas_index": 1,
            "atlas_tile_variant": 1,
            "atlas_tile": "bubble_column_mid",
            "ticks_per_frame": 1,
            "frames": [ 31, 30, 29, /* ...ละไว้... */ 2, 1, 0 ],
            "blend_frames": false
        }

3. Respawn Anchor Top (ด้านบนของ Respawn Anchor)

{
            "flipbook_texture": "textures/blocks/respawn_anchor_top",
            "atlas_tile": "respawn_anchor_top",
            "atlas_index": 2,
            "atlas_tile_variant": 2,
            "frames": [ 0, 1, 2, /* ...เรียงไปจนถึง... */ 31 ]
        }

คุณสมบัติของ Flipbook Texture

ตารางด้านล่างนี้อธิบายความหมายของค่าต่างๆ ที่เราใช้กำหนดอนิเมชันครับ:

ชื่อคุณสมบัติ (Name) คำอธิบาย (Description)
flipbook_texture ที่อยู่ไฟล์รูปภาพ (Path) สำหรับด้านใดด้านหนึ่งหรือหลายด้านของบล็อก
atlas_index หมายเลขของเฟรมแรกที่จะให้แสดงเมื่อเริ่มอนิเมชัน (และถ้าเกิดข้อผิดพลาดขึ้น เฟรมนี้จะถูกนำมาแสดงค้างไว้)
atlas_tile ชื่อของรูปภาพที่จะถูกนำมาแสดงแทน หากอนิเมชันมีปัญหา
atlas_tile_variant รูปภาพทางเลือกที่จะให้แสดงเป็น atlas tile
ticks_per_frame จำนวน "ทิค" (Tick) ของเวลาในเกมที่จะให้ผ่านไปก่อนจะเปลี่ยนเป็นเฟรมถัดไป
(1 วินาที = 20 ทิค)
frames ชุดตัวเลข (Array) ของหมายเลขเฟรม ใช้กำหนดลำดับการแสดงผลว่าจะให้เล่นเฟรมไหนก่อนหลัง

เตรียม Resource Pack สำหรับอนิเมชัน

เพื่อให้ง่ายต่อการเรียนรู้ ให้เราเริ่มต้นโดยใช้ Resource Pack "Green Dirt Block" (บล็อกดินสีเขียว) ที่ทำเสร็จแล้ว (จากบทเรียน การสร้าง Resource Pack) หรือถ้าหาไม่เจอ ก็สามารถดาวน์โหลดได้จาก Minecraft pack samples ครับ

การแก้ไขบล็อกที่มีอนิเมชันอยู่แล้ว

เพื่อให้คุ้นเคยกับวิธีการทำงานของ Texture อนิเมชัน เราจะมาเริ่มจากการแก้ไขบล็อกที่มีอนิเมชันอยู่แล้ว ปรับเปลี่ยนมันนิดหน่อย แล้วมาดูการเปลี่ยนแปลงกันครับ

1. สร้างไฟล์กำหนดค่า (JSON)

เข้าไปที่โฟลเดอร์ textures ใน My_RESOURCE_Pack ของคุณ จากนั้นสร้างไฟล์ข้อความใหม่ชื่อ flipbook_texture.json และใส่โค้ดนี้ลงไปครับ: 

[
          {
            "flipbook_texture": "textures/blocks/sculk",
            "atlas_tile": "sculk",
            "frames": [ 0, 1, 2, 3],
            "ticks_per_frame": 20
          }
        ]

(โค้ดนี้สั่งให้บล็อก Sculk เล่นเฟรมที่ 0 ถึง 3 โดยเปลี่ยนเฟรมทุกๆ 20 ทิค หรือ 1 วินาที)

2. เตรียมไฟล์รูปภาพ Texture

  1. เข้าไปที่โฟลเดอร์ blocks (ที่อยู่ใน textures)
  2. ดาวน์โหลดไฟล์ sculk.png ด้านล่างนี้ แล้วนำไปวางไว้ในโฟลเดอร์ครับ

3. แก้ไขรูปภาพ

ให้เปิดไฟล์ sculk.png ในโปรแกรมแต่งภาพ (เช่น Paint.net, Photoshop, หรือ Pixel Studio) แล้วลองแก้ไขลวดลายในแต่ละเฟรมให้ดูแตกต่างกันเล็กน้อยครับ

4. ทดสอบในเกม

เมื่อวางไฟล์ sculk.png (ตัวที่แก้แล้ว) เสร็จเรียบร้อย ให้ติดตั้ง Resource Pack เข้าเกม Minecraft แล้วลองวางบล็อก Sculk ดูครับ มันควรจะขยับเปลี่ยนรูปร่างหน้าตาตามที่คุณวาดไว้ แบบนี้:

ตอนนี้เราเริ่มเข้าใจแล้วว่า Flipbook animation ทำงานยังไงกับบล็อกที่มีอนิเมชันอยู่แล้ว หัวข้อถัดไป เรามาลองทำให้บล็อกที่ ไม่เคยมี อนิเมชันมาก่อน ขยับได้กันบ้างดีกว่าครับ!

การใส่อนิเมชันให้บล็อกปกติ: แร่ลาพิส Lapis Lazuli Ore

คราวนี้เราจะมาต่อยอดจากเดิม โดยการเพิ่มอนิเมชันให้กับบล็อกที่ปกติแล้วไม่มีอนิเมชันอย่าง "แร่ลาพิส" (Lapis Ore) กันครับ

1. อัปเดตไฟล์ JSON

ให้แทนที่เนื้อหาเดิมในไฟล์ flipbook_texture.json ด้วยโค้ดชุดใหม่นี้ครับ: 

[
          {
            "flipbook_texture": "textures/blocks/sculk",
            "atlas_tile": "sculk",
            "frames": [ 0, 1, 2, 3],
            "ticks_per_frame": 20
          },
          {
            "flipbook_texture": "textures/blocks/lapis_ore",
            "atlas_tile": "lapis_ore",
            "frames": [ 0, 1, 2, 3, 4, 5],
            "ticks_per_frame": 10
          }
        ]

อธิบายโค้ดส่วน lapis_ore:

  • "frames": [ 0, 1, 2, 3, 4, 5]: บรรทัดนี้บอกเกมว่า เรากำลังจะเพิ่มเฟรมเข้าไปอีก 5 เฟรม รวมกับของเดิม (เฟรม 0) เป็นทั้งหมด 6 เฟรม ครับ
  • "ticks_per_frame": 10: บรรทัดนี้กำหนดว่าลาพิสจะเปลี่ยนเฟรมเร็วกว่า Sculk (Sculk ใช้ 20 ทิค แต่ Lapis ใช้ 10 ทิค หรือครึ่งวินาที)

2. ปรับขนาดรูปภาพ

รูปภาพดั้งเดิมของแร่ลาพิส (Vanilla lapis ore) นั้นมีขนาด 16 x 16 พิกเซล หน้าตาแบบนี้ครับ:

แต่ในเมื่อเรากำหนดในโค้ด JSON ว่าจะมี 6 เฟรม เราจำเป็นต้องขยายความสูงของรูปภาพ (Canvas Size) เพื่อให้รองรับเฟรมทั้งหมด

สูตรคำนวณ: 16 พิกเซล (ความสูงเดิม) x 6 (จำนวนเฟรม) = 96 พิกเซล

ดังนั้น ให้เปิดไฟล์ lapis_ore.png ในโปรแกรมแต่งภาพ แล้วเปลี่ยนขนาดความสูงของ Canvas ให้เป็น 96 พิกเซล (ความกว้างคงไว้ที่ 16 เท่าเดิม)

3. วาดเฟรมเพิ่ม

ตอนนี้เรามีพื้นที่ว่างยาวๆ สำหรับ 6 เฟรมแล้ว ให้คุณวาดลวดลายเพิ่มลงไปอีก 5 เฟรมที่เหลือครับ (ตัวอย่างเช่น ทำให้แร่ลาพิสค่อยๆ กะพริบแสงวิบวับ)

4. นำไปใช้งาน

นำไฟล์ที่แก้ไขแล้วไปวางในโฟลเดอร์ textures/blocks ของ Animation Pack ของคุณ และอย่าลืมเปลี่ยนชื่อจาก lapis_ore_EDITED.png ให้กลายเป็น lapis_ore.png ด้วยนะครับ

เสร็จแล้วก็เข้าเกมไปดูผลงานบล็อกลาพิสวิบวับของคุณได้เลย!

การใส่อนิเมชันให้บล็อกลูกเต๋า

หากคุณจำบทเรียน Advanced Custom Blocks ได้ คุณน่าจะมีบล็อกลูกเต๋าสีแดงอยู่ในมือ ซึ่งเมื่อวางลงไปแล้วจะดูเหมือนลูกเต๋าจริงๆ

ในตัวอย่างนี้ เราจะมาทำให้บล็อกนั้น มีชีวิต ขึ้นมาครับ โดยการทำให้มันเปลี่ยนหน้าลูกเต๋า (Scroll) วนไปเรื่อยๆ ในขณะที่ผู้เล่นถือมันอยู่ในมือหรือใน Hotbar

ข้อสังเกต: เนื่องจากวิธีการแสดงผล Texture ของไอเท็มใน Hotbar ทำให้ทั้ง 3 ด้านที่มองเห็น จะแสดงเลขเดียวกันพร้อมกันหมด ในขณะที่มันเปลี่ยนเลขจาก 1 ไปถึง 6 ครับ

1. เตรียมไฟล์

คุณจะต้องใช้ Resource Pack และ Behavior Pack ของ Custom Die block ที่ทำเสร็จสมบูรณ์แล้ว (ต้องใช้ทั้งคู่ถึงจะทำงานได้นะครับ)

2. แก้ไข JSON

ให้ไปที่ resource pack > textures ในโฟลเดอร์งานของคุณ จากนั้นเปิดไฟล์ flipbook_texture.json (หรือสร้างใหม่ถ้ายังไม่มี) แล้วเพิ่มโค้ดนี้ลงไปครับ: 

[
          {
            "flipbook_texture": "textures/blocks/die_red",
            "atlas_tile": "die_red",
            "frames": [ 0, 1, 2, 3, 4, 5 ],
            "ticks_per_frame": 20,
            "blend_frames": false
          }
        ]

3. เปลี่ยนรูปภาพ

เดิมทีไฟล์ die_red.png ของเราเป็นแค่รูปสี่เหลี่ยมจัตุรัสรูปเดียว แต่ตอนนี้เราต้องเปลี่ยนให้มันเป็น แถบยาว (Strip) ที่มีเลข 1 ถึง 6 เรียงต่อกันลงมาครับ

ให้เข้าไปที่โฟลเดอร์ textures/blocks และแทนที่ไฟล์ die_red.png เดิม ด้วยไฟล์ด้านล่างนี้:

4. ทดสอบ

เมื่อเปิดใช้งานทั้ง Resource Pack และ Behavior Pack ในโลก Creative Mode แล้ว ให้พิมพ์คำสั่งเพื่อเสกลูกเต๋า: 

/give @s demo:die

ลองสังเกตดูที่บล็อกสีแดงในมือของคุณดูสิครับ ตอนนี้มันน่าจะกำลังเปลี่ยนตัวเลขไปเรื่อยๆ แล้ว!

อนิเมชันแบบหลายหน้า

ในหัวข้อนี้ เราจะพูดถึงเฉพาะ Command Block แบบปกติ (Vanilla) สีส้มเท่านั้นนะครับ จะไม่รวมถึงแบบ Conditional, Chain (สีเขียว), หรือพวกที่มีคำว่า mipmap ในชื่อ เพราะหลักการทำอนิเมชันของพวกมันเหมือนกันหมดครับ

ที่ผมต้องบอกไว้ก่อนก็เพื่อว่า เวลาที่คุณเข้าไปขุดคุ้ยหาโค้ดในไฟล์ flipbook_texture.json หรือหารูปภาพในโฟลเดอร์ของ Vanilla sample pack คุณจะได้มองหาชื่อไฟล์ที่ตรงกับบรรทัด "flipbook_texture": ในโค้ดได้ถูกตัวครับ

สำรวจลวดลาย

ถ้าคุณลองสังเกต Command Block แบบใกล้ๆ คุณจะเห็นว่า ด้านหน้า, ด้านหลัง, และด้านข้าง มีรูปทรงลวดลายที่แตกต่างกัน และมีจุดไฟกะพริบที่ไม่เหมือนกันด้วย

  • ด้านหน้า (Front): มีลวดลายเป็นรูปวงกลมตรงกลาง
  • ด้านข้างทั้ง 4 (Sides): มีลวดลายเป็นรูปลูกศรชี้ไปทิศทางเดียวกัน
  • ด้านหลัง (Back): มีลวดลายเป็นรูปสี่เหลี่ยม

โค้ดกำหนดค่า

เฟรมต่างๆ ถูกนำไปแปะลงบน Command Block ด้วยโค้ดชุดนี้ครับ (สังเกตว่ามีการแยก Block 3 ชุด สำหรับ 3 ส่วน): 

{
              "flipbook_texture": "textures/blocks/command_block_front",
              "atlas_tile": "command_block_front",
              "ticks_per_frame": 10
            },
            {
              "flipbook_texture": "textures/blocks/command_block_back",
              "atlas_tile": "command_block_back",
              "ticks_per_frame": 10
            },
            {
              "flipbook_texture": "textures/blocks/command_block_side",
              "atlas_tile": "command_block_side",
              "ticks_per_frame": 10
