เสียงแบบกำหนดเอง Custom Sounds

เพิ่มมิติใหม่ให้โลก Minecraft ด้วยเสียงของคุณเอง

เสียงเป็นวิธีที่ยอดเยี่ยมในการเพิ่มความรู้สึกอินและดื่มด่ำ (Immersion) ไปกับคอนเทนต์ของคุณครับ ไม่ว่าจะเป็นเสียงบล็อก, สัตว์, หรือแผ่นเพลง (Music disc) ที่แสนจะสำคัญ เสียงต่างๆ ที่คุณได้ยินขณะเล่น Minecraft จะดึงคุณให้ดำดิ่งเข้าสู่ประสบการณ์นั้นๆ

ในบทช่วยสอนนี้ เราจะมาเพิ่มเสียงแบบกำหนดเองที่จะเล่น เมื่อเราเปิดหีบสมบัติ (Chest) กันครับ ซึ่งจะช่วยเพิ่มความดื่มด่ำและสะท้อนเอกลักษณ์เฉพาะตัวให้กับโลกของคุณได้เป็นอย่างดี

สิ่งที่ควรรู้ก่อนเริ่ม

เราขอแนะนำให้คุณทำความเข้าใจบทเรียนต่อไปนี้ให้เสร็จสิ้นก่อนเริ่มบทช่วยสอนนี้นะครับ:

  • การเริ่มต้นพัฒนา Add-ons
  • ความรู้เบื้องต้นเรื่อง Resource Packs
  • ความรู้เบื้องต้นเรื่อง Behavior Packs

*นอกจากนี้ คุณจะต้องดาวน์โหลด Resource Pack (แพ็กต้นฉบับของเกม) เตรียมเอาไว้ด้วยนะครับ

สร้าง Resource Pack

การตั้งค่าโฟลเดอร์เสียง

ตอนนี้หลังจากที่สมมติว่าคุณได้สร้างโครงสร้างพื้นฐานของ Resource Pack ไว้เรียบร้อยแล้ว ก็ถึงเวลาเตรียมสถานที่สำหรับเก็บไฟล์เสียงของเรากันครับ

ให้ทำตามขั้นตอนดังนี้เพื่อเข้าไปยังโฟลเดอร์ com.mojang ใน File Explorer:

  1. กดปุ่ม Win + R บนคีย์บอร์ดเพื่อเปิดหน้าต่าง Run
  2. หากคุณใช้เกม Minecraft รุ่นปกติ ให้คัดลอกข้อความด้านล่างนี้ไปวางในช่อง Open: 
    %appdata%\Minecraft Bedrock\users\shared\games\com.mojang
    หรือถ้าคุณใช้ Minecraft รุ่นพรีวิว (Preview) ให้คัดลอกข้อความด้านล่างนี้ไปวางในช่อง Open: 
    %appdata%\Minecraft Bedrock Preview\users\shared\games\com.mojang
  3. คลิก OK
  4. เปิดเข้าไปในโฟลเดอร์ development_resource_packs
  5. เปิดเข้าไปในโฟลเดอร์แพ็กของคุณ (ในตัวอย่างนี้สมมติว่าชื่อ custom_sound_pack)
  6. สร้างโฟลเดอร์ใหม่และตั้งชื่อว่า sounds

เมื่อทำเสร็จแล้ว โครงสร้างไฟล์และโฟลเดอร์ของคุณจะมีหน้าตาประมาณนี้ครับ:

คุณจะต้องสร้างโฟลเดอร์ย่อยเพิ่มเติมข้างในนี้อีกในภายหลัง

การค้นหาเสียงที่ใช่

สถานที่เก็บไฟล์พร้อมแล้ว ตอนนี้ก็ถึงเวลาเตรียมไฟล์เสียงแบบกำหนดเองของคุณให้พร้อมครับ! ในตัวอย่างนี้ เราจะใช้โปรแกรม Audacity แต่อันที่จริงคุณสามารถใช้ซอฟต์แวร์จัดการเสียงตัวไหนก็ได้ครับ ตราบใดที่มันสามารถส่งออกไฟล์ (Export) ออกมาเป็นรูปแบบ .ogg ได้

คุณสามารถเลือกที่จะบันทึกเสียงใหม่ผ่านโปรแกรมของคุณ หรือจะเปิดไฟล์เสียงที่มีอยู่ในคอมพิวเตอร์ขึ้นมาแก้ไขก็ได้ครับ เมื่อเปิดไฟล์ในโปรแกรมและปรับแต่งจนพอใจแล้ว จะต้อง Export มันออกมาเป็นไฟล์ .ogg (หากใช้โปรแกรม Audacity สามารถทำได้โดยไปที่เมนู File > Export > Export as OGG ครับ)

สำหรับบทช่วยสอนนี้ เราจะทำการส่งออกไฟล์เสียงแบบกำหนดเองของเราโดยตั้งชื่อว่า chestopen.ogg ครับ (ส่วนเหตุผลที่ต้องตั้งชื่อนี้ เดี๋ยวเราจะได้ทราบกันในขั้นตอนต่อไปครับ)

หากคุณยังไม่มีไฟล์เสียง และต้องการไฟล์ตัวอย่างเพื่อนำไปใช้ทดลองทำตาม สามารถดาวน์โหลดได้ที่นี่ครับ:

ดาวน์โหลดไฟล์ตัวอย่าง (.ogg)

จัดหาที่อยู่ให้ไฟล์เสียง

ขอแสดงความยินดีด้วยครับ ทุกอย่างกำลังไปได้สวยเลย ตอนนี้คุณมีเสียงที่ต้องการจะแทนที่ มีไฟล์เสียงใหม่เตรียมไว้พร้อมแล้ว และตั้งค่า Resource Pack เสร็จเรียบร้อยแล้ว ถึงเวลามาดูกันว่าไฟล์เสียงของคุณควรจะไปอยู่ตรงไหนครับ

ในโฟลเดอร์ของ Vanilla Resource Pack (แพ็กต้นฉบับของเกม) จะรวบรวมไฟล์เสียงทุกอย่างใน Minecraft แบบปกติเอาไว้ครับ การเข้าไปดูในโฟลเดอร์เหล่านั้น จะเป็นตัวอย่างชั้นดีให้คุณรู้ว่าต้องจัดโครงสร้าง Resource Pack ของคุณอย่างไรถึงจะไปแทนที่เสียงที่ต้องการได้

จากจุดนี้ คุณอาจจะค่อยๆ เลื่อนหาไฟล์เสียงเปิดหีบจากรายชื่อเสียงทั้งหมดในเกมทีละไฟล์ก็ได้ แต่เราแนะนำให้ใช้ปุ่ม CTRL+F เพื่อค้นหาชื่อไฟล์จะเร็วกว่ามากครับ สำหรับตัวอย่างของเรา ไฟล์ที่เรากำลังตามหาคือไฟล์เปิดหีบสมบัติ ซึ่งบังเอิญว่ามันตั้งอยู่ใน Path: sounds/random/chestopen พอดีครับ

นำไฟล์ไปวางให้ถูกที่

ตอนนี้เรารู้แล้วว่าต้องเอาไฟล์เสียงแบบกำหนดเองไปวางไว้ที่ไหน ให้ทำตามนี้เลยครับ:

  1. เข้าไปที่ com.mojang > development_resource_packs > custom_sound_pack > sounds
  2. ภายในโฟลเดอร์ sounds ให้สร้างโฟลเดอร์ใหม่ชื่อว่า random
  3. นำไฟล์ chestopen.ogg ของคุณไปใส่ไว้ในโฟลเดอร์ random นั้นได้เลยครับ

การแทนที่เสียงอื่นๆ

หากคุณต้องการแทนที่เสียงอื่นๆ ก็แค่หาไฟล์นั้นใน Vanilla Resource Pack ให้เจอ แล้วจำลองโครงสร้างโฟลเดอร์แบบเดียวกันขึ้นมาใน Resource Pack ของคุณครับ ตัวอย่างเช่น ถ้าคุณต้องการเปลี่ยนเสียงฝนตกในโลกของคุณ คุณก็ต้องสร้างโฟลเดอร์ย่อยเรียงซ้อนกันตามนี้:

com.mojang > development_resource_packs > custom_sound_pack > sounds > ambient > weather

จากนั้นภายในโฟลเดอร์ weather คุณก็สามารถใส่ไฟล์เสียงฝนที่คุณต้องการจะนำไปแทนที่ของเดิมได้เลยครับ

⚠️ หมายเหตุ

คุณต้องตั้งชื่อไฟล์เสียงใหม่ให้ เหมือนกันทุกประการ กับไฟล์ที่คุณต้องการแทนที่ อย่างไรก็ตาม คุณ ไม่จำเป็น ต้องใช้นามสกุลไฟล์ (Format) เดียวกันครับ ดังนั้น คุณสามารถใช้ไฟล์ rain1.ogg ของคุณไปแทนที่ไฟล์ rain1.fsb ของเกมได้เลย ตราบใดที่ชื่อไฟล์ก่อนจุดนามสกุลสะกดตรงกันเป๊ะๆ ครับ

ทดสอบเสียงของคุณ

งานหนักผ่านพ้นไปแล้วครับ! ตอนนี้ก็ถึงเวลามาทดสอบเสียงแบบกำหนดเองของเรากัน ในการทดสอบเสียง ให้คุณสร้างโลกทดสอบ (Test world) ใหม่ในเกม โดยเปิดใช้งานแพ็ก custom_sound_pack ของคุณ และอย่าลืมเปิดตั้งค่า Cheats เป็น ON ไว้ด้วยนะครับ เพื่อให้เราสามารถใช้คำสั่ง /give ได้

ขั้นตอนการทดสอบ:

  1. เข้าสู่โลกที่คุณสร้างไว้ และเปิดกล่องแชท (Chat box) ขึ้นมา
  2. พิมพ์คำสั่งต่อไปนี้เพื่อเสกหีบสมบัติให้ตัวเอง: 
    /give @s chest
  3. วางหีบสมบัติลงบนพื้นแล้วลอง เปิดหีบ ดูครับ

การเปิดหีบสมบัติควรจะทำให้เสียงแบบกำหนดเองของคุณเล่นขึ้นมาทันทีครับ!

การเพิ่มเสียงใหม่

แค่เสียงฝนตก 4 แบบยังไม่จุใจใช่ไหมครับ? ถ้างั้นเรามาค้นพบวิธีเพิ่มเสียงใหม่เอี่ยมลงใน Minecraft กันเลยดีกว่า สำหรับภารกิจนี้ เราจะต้องเจาะลึกเข้าไปในไฟล์ sounds.json และ sound_definitions.json ครับ

หากคุณยังไม่มีไฟล์เหล่านี้ ให้สร้างไฟล์ตามที่อยู่ด้านล่างนี้ได้เลย:

  • สร้าง sounds.json ไว้ที่ com.mojang > development_resource_packs > custom_sound_pack
  • สร้าง sound_definitions.json ไว้ที่ com.mojang > development_resource_packs > custom_sound_pack > sounds

(คุณสามารถใช้ไฟล์ sounds.json ตัวอย่างจาก Vanilla Resource Pack เพื่อช่วยค้นหาหมวดหมู่ของเสียงที่คุณต้องการเพิ่มได้ครับ)

ทำให้หีบสมบัติสุ่มเสียงได้!

เอาล่ะ แทนที่เราจะแค่แทนที่เสียงเปิดหีบเสียงเดียวเหมือนที่ทำไปก่อนหน้านี้ สมมติว่าเราอยากให้มีเสียง 2 เสียงที่แตกต่างกัน และให้เกมสุ่มเล่น 1 เสียงเวลาที่เราเปิดหีบ

ให้กลับไปที่โฟลเดอร์ com.mojang > development_resource_packs > custom_sound_pack > sounds > random แล้วเปลี่ยนชื่อไฟล์ chestopen.ogg (ที่คุณเพิ่งใส่เข้าไป) เป็น chestopen2.ogg ครับ (แล้วอย่าลืมหาไฟล์ chestopen.ogg อีกเสียงนึงมาใส่คู่กันด้วยนะครับ)

1. ตั้งค่าไฟล์ sounds.json

เปิดไฟล์ sounds.json ขึ้นมาแล้วเพิ่มโค้ดต่อไปนี้ลงไปครับ:

{
            "individual_event_sounds" : {
                "events" : {
                    "chest.open" : {
                        "pitch" : [ 0.90, 1.0 ],
                        "sound" : "random.chestopen",
                        "volume" : 0.50
                    }
                }
            }
        }

2. ตั้งค่าไฟล์ sound_definitions.json

เปิดไฟล์ sound_definitions.json ขึ้นมาแล้วเพิ่มโค้ดนี้ลงไปครับ (สังเกตว่าใน array "sounds" เราใส่ไว้ทั้ง 2 ไฟล์เลย): 

{
            "format_version" : "1.20.20",
            "sound_definitions" : {
                "random.chestopen" : {
                    "__use_legacy_max_distance" : "true",
                    "category" : "block",
                    "max_distance" : null,
                    "min_distance" : null,
                    "sounds" : [ 
                        "sounds/random/chestopen",
                        "sounds/random/chestopen2"
                    ]
                }
            }
        }

เมื่อเราบันทึกไฟล์แล้วเข้าไปทดสอบเสียงกันอีกครั้งในเกม คุณจะสังเกตเห็นว่าการเปิดหีบ จะสุ่มเล่นเสียงใดเสียงหนึ่งจาก 2 เสียงที่คุณกำหนดไว้ครับ!

การใช้คำสั่ง /playsound

นอกจากจะผูกเสียงเข้ากับเหตุการณ์ในเกม (In-game events) แล้ว คุณยังสามารถเพิ่มเสียงแบบกำหนดเองที่ไม่ได้เล่นตามเหตุการณ์ใดๆ ได้ด้วยครับ

คุณสามารถใช้คำสั่ง /playsound เพื่อเล่นเสียงใดๆ ก็ตามในเกม รวมถึงเสียงที่เพิ่มเข้ามาใหม่ใน Resource Pack ของคุณด้วย แม้ว่าเสียงเหล่านั้นจะไม่ได้ถูกตั้งให้เล่นอัตโนมัติ แต่คุณก็ยังสามารถพิมพ์คำสั่งสั่งเล่นมันด้วยตัวเอง หรือจะประยุกต์ใช้ Command blocks เพื่อสั่งเล่นเสียงในเวลาที่เหมาะสม ตามสถานการณ์ในโลกหรือโหมดเกมของคุณก็ได้ครับ

หากต้องการเรียนรู้เพิ่มเติมเกี่ยวกับคำสั่ง /playsound คุณสามารถเข้าไปอ่านรายละเอียดได้ที่ เอกสารอ้างอิงคำสั่ง ครับ
