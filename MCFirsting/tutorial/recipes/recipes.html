การสร้างสูตรคราฟต์ Recipes

ปรุงแต่งจินตนาการให้กลายเป็นไอเท็มจริง

สูตรการคราฟต์ (Recipes) ใน Minecraft นั้นมีความคล้ายคลึงกับสูตรอาหารในชีวิตจริงมากครับ คือเราต้องรวบรวมส่วนประกอบต่างๆ เตรียมพวกมันด้วยวิธีที่เฉพาะเจาะจง และในที่สุดเราก็จะได้สิ่งใหม่ที่ยอดเยี่ยมออกมา

ข้อแตกต่างหลักคือใน Minecraft สูตรเหล่านี้ไม่ได้จำกัดอยู่แค่เรื่องอาหารการกินเท่านั้น แต่เรายังใช้มันเพื่อสร้างเฟอร์นิเจอร์, ทรัพยากรที่ซับซ้อน, และสิ่งอื่นๆ อีกมากมาย!

ในบทความนี้ เราจะมาดูส่วนประกอบพื้นฐานของสูตรคราฟต์ และทำความรู้จักกับสูตรประเภทต่างๆ เพื่อช่วยให้คุณเริ่มต้นใช้งานได้ทันทีครับ

พารามิเตอร์ของสูตร Recipe Parameters

พารามิเตอร์ของสูตรเปรียบเสมือนบล็อกพื้นฐานที่ประกอบกันขึ้นมาเป็นสูตรคราฟต์ครับ ถ้าจะเปรียบเทียบกับเรื่องอาหารเมื่อครู่ สิ่งเหล่านี้ก็คือ "ส่วนผสม" ของเรานั่นเอง! แม้ว่าทุกสูตรจะมีพารามิเตอร์ แต่ก็ไม่ใช่ทุกสูตรที่จะใช้พารามิเตอร์ชุดเดียวกันนะครับ

ชื่อ คำอธิบาย ประเภทที่ใช้
Input กำหนดไอเท็มวัตถุดิบที่ใช้ในการสร้าง Furnace; Potion
Output กำหนดไอเท็มผลลัพธ์ที่จะได้จากสูตรนั้นๆ Furnace; Potion
Tags กำหนดกลุ่มของไอเท็ม (Tag) ที่สามารถใช้ในสูตรนี้ได้ All (ทั้งหมด)
Pattern ตัวแทนของไอเท็มและการจัดวางรูปแบบที่จำเป็นเพื่อให้ได้ผลลัพธ์ตามต้องการ Shaped
Key กำหนดไอเท็ม Minecraft ให้จับคู่กับตัวอักษร (Key) ที่คุณใช้ใน Pattern Shaped
Priority กำหนดระดับความสำคัญของสูตร (ตัวเลขยิ่งต่ำ ยิ่งมีความสำคัญสูง) Shaped; Shapeless
Result เมื่อใส่วัตถุดิบถูกต้องตาม Pattern ไอเท็มเหล่านี้คือผลลัพธ์ที่จะออกมา Shaped; Shapeless; Smithing Transform
Ingredients กำหนดไอเท็มที่ต้องใช้ในสูตร (โดยไม่สนใจตำแหน่งการวางในตารางคราฟต์) Shapeless
Reagent กำหนดไอเท็มที่จะนำมาผสมกับ Potion ในเครื่องปรุงยา (Brewing Stand) Potion
Addition กำหนดไอเท็มที่ใช้เสริมเพื่อทำการแปลงสภาพ (Transformation) Smithing Transform
Base กำหนดไอเท็มหลักที่จะถูกแปลงสภาพ Smithing Transform

แท็กของสูตร Recipe Tags

Recipe tags ช่วยให้คุณสามารถจัดกลุ่มองค์ประกอบที่คล้ายคลึงกันเข้าด้วยกันได้ครับ

ตัวอย่างเช่น คุณสามารถรวมบันไดไม้ชนิดต่างๆ เข้าด้วยกันเพื่อสร้างสูตรคราฟต์ที่สามารถปรับเปลี่ยนไปตามพารามิเตอร์ที่หลากหลายได้ (กล่าวคือ สามารถใช้วัตถุดิบในกลุ่มเดียวกันชนิดไหนก็ได้มาใช้แทนกันในสูตร)

คุณสามารถดูข้อมูลเพิ่มเติมเกี่ยวกับ Recipe tags ได้ที่ บทความนี้ (ภาษาอังกฤษ) ครับ

ประเภทของสูตร Recipe Types

1. Shaped vs. Shapeless Recipes

Shaped Recipes (สูตรแบบมีรูปร่าง)

ผู้เล่นจะใช้ Shaped Recipes บนโต๊ะคราฟต์ (Crafting Table) โดยต้องวาง ส่วนประกอบ (Components) ของสูตรลงในตำแหน่งตามรูปแบบ (Pattern) ที่กำหนดเป๊ะๆ เพื่อสร้างผลลัพธ์ที่ต้องการ ในฐานะผู้สร้าง คุณสามารถใช้ Resource และ Behavior packs เพื่อปรับแต่งการจัดวางและผลลัพธ์ให้เหมาะกับโลกของคุณได้ครับ

📝 หมายเหตุ

ขนาดสูตรสูงสุดที่รองรับคือ 3x3 ดังนั้นอะไรก็ตามที่เกินจากตัวอักษรที่ 3 ใน Pattern จะถูกมองข้ามไปครับ

Shapeless Recipes (สูตรแบบไร้รูปร่าง)

Shapeless recipes คือสูตรที่ไม่ต้องการรูปแบบการวางที่ตายตัว ขอแค่มีวัตถุดิบครบก็พอ คุณสามารถสร้างหรือแก้ไขสูตรเหล่านี้ได้เช่นกัน แต่พารามิเตอร์ที่ใช้จะแตกต่างกันเล็กน้อยครับ

ตารางเปรียบเทียบพารามิเตอร์ (Shaped vs Shapeless)

ชื่อ คำอธิบาย ประเภท
Tags กำหนดไอเท็มที่ใช้ในสูตรได้ (กลุ่ม) ทั้งคู่
Pattern ตัวแทนไอเท็มและการจัดวางรูปแบบ Shaped
Key จับคู่ตัวอักษรใน Pattern กับไอเท็ม Minecraft Shaped
Ingredients กำหนดไอเท็มที่ต้องใช้ (ไม่สนตำแหน่งการวาง) Shapeless
Priority ระดับความสำคัญ (เลขยิ่งน้อย ยิ่งสำคัญมาก) ทั้งคู่
Result ผลลัพธ์ที่จะได้เมื่อใส่วัตถุดิบถูกต้อง ทั้งคู่

2. สูตรเตาเผา Furnace Recipes

คือสูตรที่ต้องใช้ เตาเผา (Furnace) ในการสร้างผลลัพธ์ เมื่อผู้เล่นใส่ไอเท็มที่ถูกต้องลงไป (ตามที่คุณกำหนดเป็น Input ในไฟล์ JSON) มันจะเปลี่ยนสภาพกลายเป็น Output ที่กำหนดไว้ครับ

ชื่อ คำอธิบาย
Tags กำหนดไอเท็มที่ใช้ในสูตรได้
Input กำหนดไอเท็มวัตถุดิบที่จะนำมาเผา
Output กำหนดไอเท็มผลลัพธ์ที่ได้จากการเผา

3. สูตรปรุงยา Potion Brewing Recipes

การปรุงยาต้องใช้ทั้ง Potion Brewing Mix (ส่วนผสมยา) และ Potion Brewing Container (ภาชนะใส่ยา) ซึ่งทำที่แท่นปรุงยา (Brewing Station) คุณสามารถปรับแต่งพารามิเตอร์เหล่านี้ได้ผ่าน Resource และ Behavior packs ครับ

ชื่อ คำอธิบาย
Tags กำหนดไอเท็มที่ใช้ในสูตรได้
Input กำหนดไอเท็มตั้งต้น (เช่น ขวดน้ำ หรือยาพื้นฐาน)
Reagent กำหนดไอเท็มที่จะนำมาผสม (ตัวทำปฏิกิริยา) ใส่ด้านบนของแท่นปรุงยา
Output กำหนดไอเท็มผลลัพธ์ที่ได้

4. สูตรการแปลงสภาพ Smithing Transform Recipes

ผู้เล่นใช้สูตรนี้ที่ โต๊ะตีเหล็ก (Smithing Table) เพื่อแปลงสภาพไอเท็มให้กลายเป็นอีกอย่างหนึ่ง โดยที่ ยังคงคุณสมบัติเดิมของไอเท็มนั้นไว้ (เช่น การอัปเกรดเกราะ Diamond เป็น Netherite โดยที่ Enchantment ไม่หาย)

ชื่อ คำอธิบาย
Base กำหนดไอเท็มหลักที่จะถูกแปลงสภาพ
Addition กำหนดไอเท็มเสริมที่ใช้ในการแปลงสภาพ (เช่น Netherite Ingot)
Result ผลลัพธ์ที่จะได้เมื่อใส่วัตถุดิบครบถ้วน

5. สูตรตกแต่งลวดลาย Smithing Trim Recipes

ผู้เล่นใช้สูตรนี้ที่ โต๊ะตีเหล็ก (Smithing Table) เช่นกัน แต่เพื่อจุดประสงค์ในการเพิ่ม ลวดลายสีสัน (Color Trim) ให้กับไอเท็ม โดยที่ยังคงคุณสมบัติเดิมของไอเท็มนั้นไว้ครับ

การปลดล็อกสูตร Recipe Unlocking

ตั้งแต่เวอร์ชัน 1.20.10 เป็นต้นมา ผู้เล่นสามารถ "ปลดล็อก" สูตรคราฟต์ได้ครับ ซึ่งมีตั้งแต่ของพื้นฐาน (เช่น พลั่วไม้) ไปจนถึงไอเท็มที่ซับซ้อนและทรงพลังที่สุดใน Minecraft (เช่น แร่เพชร หรือแอปเปิ้ลทองชาร์จ)!

ในฐานะส่วนหนึ่งของกระบวนการออกแบบไอเท็มและสูตร คุณสามารถกำหนดเงื่อนไขในการปลดล็อกไอเท็มได้โดยใช้เองค์ประกอบ unlock ในไฟล์ JSON ของสูตรครับ

💡 เคล็ดลับ

คุณสามารถเลือกกลยุทธ์การปลดล็อกสูตรได้ 2 แบบครับ คือ 1. การค้นพบไอเท็ม หรือ 2. บริบทเฉพาะในโลกเกม

ตัวอย่างการปลดล็อก

1. ปลดล็อกด้วยไอเท็ม

เราใช้ item เพื่อระบุไอเท็มที่จะทำให้เกิดการปลดล็อก และสามารถเพิ่มแท็ก data เพื่อกรองชนิดของไอเท็มให้เจาะจงลงไปได้ครับ

ตัวอย่าง: ไม้แปรรูปอะคาเซีย (Acacia Planks)

"unlock": [
            {
              "item": "minecraft:wood",
              "data": 4
            }
          ]

(ความหมาย: เมื่อผู้เล่นเก็บไม้ Acacia ได้ สูตร Acacia Planks จะถูกปลดล็อก)

2. ปลดล็อกด้วยบริบท (Context)

การใช้ context จะขึ้นอยู่กับสถานการณ์ของผู้เล่น โดยมีค่าที่ใช้ได้ 3 แบบดังนี้:

  • AlwaysUnlocked: ปลดล็อกทันทีเสมอ (เหมาะสำหรับสูตรพื้นฐานที่ใครๆ ก็ควรรู้)
  • PlayerInWater: ปลดล็อกเมื่อผู้เล่นลงน้ำครั้งแรก
  • PlayerHasManyItems: ปลดล็อกเมื่อผู้เล่นมีไอเท็มในตัวมากกว่า 10 ชิ้น

ตัวอย่าง: โต๊ะคราฟต์ (Crafting table)

"unlock": {
            "context": "AlwaysUnlocked"
          }

ตัวอย่าง: หีบสมบัติ (Chest)

"unlock": {
            "context": "PlayerHasManyItems"
          }
