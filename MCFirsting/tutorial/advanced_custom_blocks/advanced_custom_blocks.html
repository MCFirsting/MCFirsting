การสร้างบล็อกขั้นสูง Advanced Custom Blocks

ก้าวข้ามขีดจำกัดของบล็อกสี่เหลี่ยม ด้วยรูปทรงและโมเดลที่คุณออกแบบเองได้

ในบทเรียนนี้ คุณจะได้เรียนรู้เจาะลึกเกี่ยวกับองค์ประกอบ (Components) ของ Custom Block ให้มากขึ้น ซึ่งรวมถึง:

  • การใช้โปรแกรม Blockbench เพื่อออกแบบรูปทรง (Geometry) และลวดลาย (Texture)
  • การเพิ่มรูปทรงของบล็อกที่มีรูปร่างหรือขนาดแตกต่างไปจากสี่เหลี่ยมปกติ (16x16x16)
  • การใช้คอมโพเนนต์ minecraft:geometry เพื่ออ้างอิงรูปทรงที่เราสร้างขึ้นเอง
  • การใช้คอมโพเนนต์ minecraft:material_instances เพื่อกำหนด Texture ให้กับรูปทรงนั้น

คำแนะนำ:

หากคุณยังไม่ได้เรียนบทเรียน วิธีสร้าง Custom Block เบื้องต้น ขอแนะนำให้กลับไปศึกษาและทำให้สำเร็จก่อนเริ่มบทเรียนนี้ครับ

ก่อนหน้านี้เราได้สร้าง "บล็อกลูกเต๋า" กันไปแล้ว ซึ่งเป็นวิธีที่ใช้ไฟล์และเขียน JSON น้อยที่สุดเท่าที่จะทำได้ แต่คราวนี้ เราจะมาสำรวจความเป็นไปได้ใหม่ๆ ที่น่าตื่นเต้นยิ่งกว่าเดิม โดยใช้ JSON components ใน Behavior Pack ที่ชื่อว่า geometry และ material_instances กันครับ!

โครงสร้างไฟล์ของคุณควรจะเป็นแบบนี้:

บล็อกซูชิ Sushi Blocks

ในโปรเจกต์นี้ เราจะมาสร้างบล็อกแบบกำหนดเองโดยใช้ Geometry และ Material Instances ซึ่งผลลัพธ์ที่เราจะทำกันก็คือ... ซูชิ (Sushi) ครับ!

มาเริ่มใช้ Blockbench

ในบทเรียนนี้ เราจะต้องลงมือทำงานกันในโปรแกรม Blockbench ครับ ซึ่ง Blockbench เป็นโปรแกรมสร้างโมเดล 3D ฟรี สำหรับสร้างโมเดล Entity ใน Minecraft: Bedrock Edition โดยเฉพาะ

นอกจากจะปั้นโมเดลได้แล้ว ผู้ใช้ยังสามารถสร้าง Texture และทำอนิเมชันสำหรับแผนที่ Bedrock Edition ได้อีกด้วย Blockbench ยังรองรับระบบปลั๊กอินภายนอก ซึ่งช่วยให้นักพัฒนาสามารถสร้างเครื่องมือเสริมเพื่อนำมาใช้งานภายในโปรแกรมได้ และยังสามารถส่งออก (Export) โมเดลเป็นไฟล์มาตรฐานแบบ OBJ, โมเดลบล็อกของ Java Edition, และรูปแบบอื่นๆ ได้อีกมากมาย

เราจะใช้ Blockbench ในบทเรียนชุดนี้เพื่อสร้าง Geometry สำหรับบล็อกขั้นสูงต่างๆ ที่เราจะทำกัน ได้แก่ ซูชิ, ปลาในฟองน้ำ และ ร่มกันแดดยักษ์

การจัดการรูปทรงใน Geometry

แต่ละ Custom Block จะมีค่า Unit Vector (UV) ที่แตกต่างกันสำหรับ Texture ในแต่ละด้านของซูชิ ค่า UV จะเป็นตัวกำหนดว่ารูปทรงเรขาคณิต (Geometry) จะถูกวางทาบลงบน Texture อย่างไร

สาหร่ายที่ใช้ห่อจะเหมือนกันทั้ง 3 แบบ (ที่เราจะสร้างกัน) ดังนั้นคุณสามารถใช้ Geometry เดิมซ้ำได้เลย!

อย่างไรก็ตาม Texture ของไส้ซูชิแต่ละชิ้นจะแตกต่างกันไปตามชนิดของโรล นี่คือตัวอย่างของ "แซลมอนโรล" (Salmon roll) ครับ:

เราขอแนะนำอย่างยิ่งให้คุณเรียนรู้การใช้งาน Blockbench อย่างละเอียด หากคุณเริ่มสร้าง Entity ที่ซับซ้อนและมีเอกลักษณ์เฉพาะตัวมากขึ้น แต่สำหรับบทเรียนนี้ เราจะใช้วิธีลัดโดยการ สร้างไฟล์เปล่าๆ ขึ้นมา แล้ว Export ลงเครื่องคอมพิวเตอร์

การส่งออกไฟล์จาก Blockbench

กรณีหากคุณลองวิธีลัดนำโค้ดโมเดลทั้งหมดมาใช้ ไห้สร้างไฟล์ .json ไว้ล่วงหน้า (สร้างไฟล์เปล่า) ให้เปิดไฟล์ .json ที่คุณเพิ่งสร้างมา แล้วนำโค้ดด้านล่างนี้ไปวางทับได้เลยครับ นี่คือข้อมูลรูปทรงของ "ซูชิ" ที่เราเตรียมไว้ให้: 

{ 
          "format_version": "1.12.0", 
          "minecraft:geometry": [ 
            { 
              "description": { 
                "identifier": "geometry.sushi", 
                "texture_width": 36, 
                "texture_height": 18, 
                "visible_bounds_width": 2, 
                "visible_bounds_height": 2.5, 
                "visible_bounds_offset": [0, 0.75, 0] 
              }, 
              "bones": [ 
                { 
                  "name": "bb_main", 
                  "pivot": [0, 0, 0], 
                  "cubes": [ 
                    {"origin": [-6, 0, -3], "size": [12, 12, 6], "uv": [0, 0]} 
                  ] 
                } 
              ] 
            } 
          ] 
        }

ตอนนี้คุณสามารถสร้าง Texture ใน Blockbench ได้เลย หรือจะสร้างในโปรแกรมแต่งภาพแยกต่างหากก็ได้ (แต่ต้องแน่ใจว่าขนาดและ UV ของรูปภาพตรงกับที่กำหนดไว้ใน Geometry นะครับ)

การตั้งค่า Resource Pack

1. เพิ่มไฟล์โมเดล Geometry

เริ่มจากการนำไฟล์โมเดลซูชิไปใส่ใน Resource Pack กันครับ

  1. เข้าไปที่โฟลเดอร์ custom_block_resource_pack ของคุณ
  2. สร้างโฟลเดอร์ชื่อ models
  3. เข้าไปใน models แล้วสร้างโฟลเดอร์ชื่อ blocks
  4. คัดลอกไฟล์ Geometry (.json) ที่ได้จาก Blockbench มาวางไว้ในโฟลเดอร์ models/blocks นี้

2. ตั้งชื่อบล็อก

เช่นเดียวกับตอนทำบล็อกลูกเต๋า เราต้องกำหนดชื่อที่แสดงในเกมให้ซูชิทั้ง 3 แบบ ให้เปิดไฟล์ texts/en_US.lang แล้วเพิ่มโค้ดนี้ต่อท้ายครับ: 

tile.demo:salmon_roll.name=Salmon Roll
tile.demo:tuna_roll.name=Tuna Roll
tile.demo:california_roll.name=California Roll

*ระวัง: อย่าเผลอเคาะเว้นวรรค (Space) หลังเครื่องหมายเท่ากับ (=) นะครับ ไม่งั้นชื่อในเกมจะแหว่งไป

3. เพิ่มรูปภาพ Textures

ให้นำไฟล์รูปภาพ Texture ของคุณไปวางไว้ในโฟลเดอร์ textures/blocks
(สำหรับบทเรียนนี้คุณสามารถบันทึกรูปตัวอย่างได้ครับ)

4. Terrain Texture

ขั้นตอนนี้คือการจับคู่ "ชื่อเรียก Texture" เข้ากับ "ที่อยู่ไฟล์ภาพ" ให้เปิดไฟล์ textures/terrain_texture.json แล้วเพิ่มโค้ดลงไปในส่วน texture_data ดังนี้: 

{ 
          "texture_data": { 
            "sushi_wrap": { 
              "textures": "textures/blocks/sushi_wrap" 
            }, 
            "salmon_roll": { 
              "textures": "textures/blocks/salmon_roll" 
            }, 
            "tuna_roll": { 
             "textures": "textures/blocks/tuna_roll" 
            }, 
            "california_roll": { 
             "textures": "textures/blocks/california_roll" 
            } 
          }
        }

5. เพิ่มเสียง blocks.json - ไม่บังคับ

การเพิ่มเสียงใน blocks.json เป็นทางเลือกเสริม เราจะให้ซูชิใช้เสียงเหมือน "โคลน" (Mud) เพราะมันนุ่มนิ่ม!

สำคัญ! เนื่องจากเรากำลังใช้ Geometry และ Material Instances ขั้นสูง ข้อมูลการ Render (แสดงผล) ในไฟล์นี้จะถูกเขียนทับโดย Behavior Pack ดังนั้นในไฟล์นี้เราจะใส่แค่ข้อมูล เสียง เท่านั้นครับ

เปิดไฟล์ blocks.json (ที่รากของ Resource Pack) แล้วเพิ่มข้อมูลต่อจาก demo:die:

{ 
          "format_version": "1.21.40", 
          "demo:die": {
          }, 
          "demo:salmon_roll": { 
            "sound": "mud" 
          }, 
          "demo:tuna_roll": { 
            "sound": "mud" 
          }, 
          "demo:california_roll": { 
            "sound": "mud" 
          }
        }

Behavior Pack

มานำชิ้นส่วนทรัพยากรทั้งหมดมารวมกันให้เป็นส่วนผสมที่ลงตัวในไฟล์ Behavior Pack กันครับ

1. สร้างไฟล์ Custom Block แยกตามชนิด

ถึงแม้ว่าบล็อกซูชิทั้ง 3 แบบจะใช้โมเดล geometry.sushi ตัวเดียวกัน แต่เราต้องสร้างไฟล์แยกสำหรับแต่ละโรล เพื่อให้มันกลายเป็นบล็อกที่มีคุณสมบัติและลวดลายเฉพาะตัวครับ

  1. เข้าไปที่โฟลเดอร์ custom_block_behavior_pack ของคุณ
  2. เข้าไปในโฟลเดอร์ blocks และสร้างไฟล์ข้อความชื่อ salmon_roll.json
  3. เปิดไฟล์ salmon_roll.json ด้วย Text Editor

2. เขียนโค้ด JSON

ในไฟล์นี้ เราจะต้องระบุว่าบล็อกนี้คืออะไร และมีหน้าตาอย่างไร ให้คัดลอกโค้ดนี้ไปวางครับ: 

{
          "format_version": "1.21.80",
          "minecraft:block": {
            "description": {
              "identifier": "demo:salmon_roll"
            },
            "components": {
              "minecraft:geometry": "geometry.sushi",
              "minecraft:material_instances":{
                "north": { 
                  "texture": "salmon_roll"
                },
                "south": { 
                  "texture": "salmon_roll"
                },
                "*": {
                  "texture": "sushi_wrap"
                }
              }
            }
          }
        }

3. เจาะลึก Component

นี่คือช่วงเวลาสำคัญที่เราจะได้ใช้พลังของ Custom Block Components ครับ!

minecraft:geometry

ใช้กำหนดว่าบล็อกนี้จะใช้โมเดลรูปร่างแบบไหน โดยเราจะอ้างอิงชื่อ Identifier (geometry.sushi) ที่เรากำหนดไว้ในไฟล์ JSON

minecraft:material_instances

คือตัวกำหนด Texture และการแสดงผลแสงเงาต่างๆ

  • การกำหนดด้าน: เราสามารถระบุชื่อด้านที่ต้องการได้ เช่น north (เหนือ) และ south (ใต้) เพื่อให้โชว์ไส้แซลมอน
  • เครื่องหมายดอกจัน (*): นี่คือกุญแจพิเศษที่หมายถึง "ด้านที่เหลือทั้งหมด" หากเราไม่ได้ระบุ Texture ให้ด้านไหนเป็นพิเศษ เกมจะดึง Texture จากตรงนี้ไปใช้ (ในที่นี้คือ sushi_wrap หรือสาหร่ายห่อ)
ข้อควรจำ: คุณ ต้อง ใส่คีย์ * ทุกครั้งเมื่อใช้งาน material_instances เพื่อกำหนดค่าเริ่มต้นครับ

ทางเลือกเสริม: การใช้ตัวแปร

เราสามารถเขียนโค้ดให้สั้นลงและเป็นระเบียบขึ้นได้ โดยการนิยาม "ชื่อตัวแปร" ให้กับ Texture ชุดนั้นๆ มาดูตัวอย่างการใช้ตัวแปร sushi_side กันครับ: 

{
          "format_version": "1.21.80",
          "minecraft:block": {
            "description": {
              "identifier": "demo:salmon_roll"
            },
            "components": {
              "minecraft:geometry": "geometry.sushi",
              "minecraft:material_instances": {
                "north": "sushi_side",
                "south": "sushi_side",
                "*": {
                  "texture": "sushi_wrap"
                },
                "sushi_side": {
                  "texture": "salmon_roll"
                }
              }
            }
          }
        }

สังเกตว่าเรากำหนดให้ sushi_side มีค่าเท่ากับ Texture salmon_roll จากนั้นในส่วน north และ south เราก็แค่พิมพ์คำว่า "sushi_side" ลงไปแทนที่จะต้องเขียน Object เต็มๆ ซ้ำหลายรอบ

ตาคุณแล้ว!

ตอนนี้ให้คุณลองสร้างไฟล์สำหรับ Tuna Roll และ California Roll ด้วยตัวเองดูครับ โดยการคัดลอกไฟล์ salmon_roll.json แล้วเปลี่ยน:

  • identifier ให้เป็นชื่อใหม่ (เช่น demo:tuna_roll)
  • เปลี่ยนชื่อ Texture ใน material_instances ให้ตรงกับชนิดของโรลนั้นๆ

การทดสอบบล็อก

เมื่อเรากำหนดค่าบล็อกซูชิทั้งใน Behavior Pack และ Resource Pack เสร็จเรียบร้อยแล้ว ก็ถึงเวลาทดสอบในเกมดูว่าหน้าตาจะเป็นอย่างไร

  1. เปิดช่องแชทขึ้นมา (กดปุ่ม T หรือ Enter บนคอมพิวเตอร์)
  2. พิมพ์คำสั่งต่อไปนี้เพื่อเสกบล็อกออกมา:
/give @s demo:salmon_roll
/give @s demo:tuna_roll
/give @s demo:california_roll

ตอนนี้คุณสามารถวางบล็อกซูชิได้ทั่วเลย! ข้อสังเกตคือ ด้วยรูปทรง (Geometry) ที่เล็กกว่าบล็อกสี่เหลี่ยมปกติ ทำให้เวลาวางเรียงกัน พวกมันจะดูเหมือนวางห่างกันนิดหน่อย ไม่ได้ชิดติดกันเป็นก้อนเดียวเหมือนบล็อกดินหรือหินครับ

อย่าลืมลองทดสอบเสกซูชิให้ครบทั้ง 3 แบบนะครับ (Salmon, Tuna, California) เพื่อชื่นชมความน่ารักของบล็อกใหม่เหล่านี้!

บทสรุป

ยินดีด้วยครับ! ในบทเรียนนี้คุณได้ก้าวข้ามขีดจำกัดของบล็อกสี่เหลี่ยมธรรมดา โดยการเรียนรู้วิธี:

  • นำโมเดล 3D จาก Blockbench มาใช้เป็น Custom Block
  • ใช้ Geometry เพื่อกำหนดรูปทรงที่ซับซ้อน
  • ใช้ Material Instances เพื่อเปลี่ยนลาย Texture บนโมเดลเดิม (โมเดลเดียวทำได้ตั้ง 3 เมนู!)

ความรู้เหล่านี้เป็นพื้นฐานสำคัญในการสร้างของตกแต่งบ้าน เฟอร์นิเจอร์ หรือไอเท็มเจ๋งๆ ในโลก Minecraft ของคุณครับ
