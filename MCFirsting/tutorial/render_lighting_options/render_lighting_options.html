ตัวเลือกการแสดงผลและแสงเงา Render & Lighting Options

เจาะลึก Material Instances เพื่อสร้างกระจกใส แสงสว่าง และเอฟเฟกต์สุดล้ำ

เอาล่ะครับ! ตอนนี้เราทำบล็อกที่มีรูปร่าง (Shape) แตกต่างกันได้แล้ว ซึ่งมันเจ๋งสุดๆ ไปเลย! แต่ถ้าเราอยากได้ลูกเล่นที่เจ๋งกว่านี้ล่ะ? เคยสงสัยไหมครับว่า "บล็อกกระจกสี" (Stained glass block) มันทำงานยังไงกันแน่?

ในบทเรียนนี้ เราจะมาสำรวจเอฟเฟกต์สนุกๆ โดยใช้ ตัวเลือกการแสดงผล (Render Options) ที่มีอยู่ในคอมโพเนนต์ material_instances กันครับ

สิ่งที่ต้องเตรียมตัวก่อนเริ่ม

ขอแนะนำให้คุณเรียนรู้และทำบทเรียนเหล่านี้ให้เสร็จก่อนที่จะเริ่มบทนี้นะครับ:

  • การสร้าง Custom Block
  • การสร้างบล็อกขั้นสูง (Advanced Custom Blocks)

ในบทเรียนนี้ เราจะเน้นทำความเข้าใจว่า วิธีการแสดงผล (Render Methods) แบบต่างๆ ในคอมโพเนนต์ material_instances ส่งผลต่อการแสดงผลบล็อกของเราอย่างไร

เพื่อให้เห็นภาพชัดเจน เราจะใช้ บล็อกปลาในฟองอากาศ (Bubble Fish) ที่แสดงอยู่ด้านล่างนี้เป็นตัวอย่างครับ เพราะบล็อกนี้มีประเภทของพิกเซลที่หลากหลาย ซึ่งจะแสดงผลแตกต่างกันไปตาม Render Method ที่เราเลือกใช้

เจ้าปลาในฟองอากาศนี้เกิดจากการประกอบกันของชิ้นส่วนหลายชิ้น:

  • ฟองอากาศ: คือบล็อกขนาด 16x16x16 (เต็มบล็อกพอดี)
  • ตัวปลาข้างใน: เป็นบล็อกขนาด 3x4x5
  • หางปลา: เกิดจากบล็อกเล็กๆ ขนาด 1x1x1 จำนวน 5 ชิ้นต่อกัน

รูปทรง (Geometry) ทั้งหมดถูกจัดวางบน UV Map เรียบร้อยแล้ว เพื่อให้แสดงผลในเกมได้อย่างถูกต้องตามที่เราต้องการ

เรื่องของสีและความโปร่งใส

Texture ของฟองอากาศถูกสร้างขึ้น (ในตัวอย่างใช้โปรแกรม Paint3D) เพื่อให้มีพิกเซลครบทั้ง 3 ประเภท ได้แก่:

  • ทึบแสง (Opaque): มองไม่ทะลุเลย (เช่น ตัวปลา)
  • โปร่งแสง (Translucent): มองทะลุได้ลางๆ เหมือนกระจกฝ้า (เช่น ผิวฟองอากาศ)
  • โปร่งใส (Transparent): มองทะลุได้ 100% (ส่วนว่างรอบๆ)

คุณสามารถสร้างบล็อกโปร่งใสในแบบของคุณเองได้ หรือจะเข้าไปดาวน์โหลดไฟล์ตัวอย่างและดูข้อมูลเพิ่มเติมได้ที่หน้า minecraft-samples ครับ

การตั้งค่า Resource Pack

นี่ไม่ใช่ครั้งแรกที่เราสร้าง Custom Blocks กัน ดังนั้นคราวนี้เราจะไปกันแบบรวบรัดนะครับ หากคุณงงตรงไหน สามารถย้อนกลับไปดูบทเรียน Custom Block ก่อนหน้านี้ได้เลยครับ

  1. ส่งออกไฟล์ (Export): ส่งออกไฟล์ Geometry จาก Blockbench เหมือนกับที่เราทำกับบล็อกซูชิ
  2. จัดเก็บไฟล์: นำไฟล์ Geometry ที่ได้ ไปใส่ในโฟลเดอร์ custom_block_resource_pack
    /models/blocks     (หรือใช้โค้ด JSON ด้านล่างนี้) 
{
          "format_version": "1.12.0",
          "minecraft:geometry": [
            {
              "description": {
                "identifier": "geometry.bubble_fish",
                "texture_width": 64,
                "texture_height": 64,
                "visible_bounds_width": 2,
                "visible_bounds_height": 2.5,
                "visible_bounds_offset": [0, 0.75, 0]
              },
              "bones": [
                {
                  "name": "bb_main",
                  "pivot": [0, 0, 0],
                  "cubes": [
                    { "origin": [-8, 0, -8], "size": [16, 16, 16], "uv": [0, 0] }
                  ]
                },
                {
                  "name": "fish",
                  "pivot": [0, -14, 0],
                  "cubes": [
                    { "origin": [-4, 6, -1], "size": [5, 4, 3], "uv": [0, 34] },
                    { "origin": [1, 7, 0], "size": [1, 1, 1], "uv": [0, 41] },
                    { "origin": [2, 8, 0], "size": [1, 1, 1], "uv": [0, 41] },
                    { "origin": [3, 9, 0], "size": [1, 1, 1], "uv": [0, 41] },
                    { "origin": [3, 5, 0], "size": [1, 1, 1], "uv": [0, 41] },
                    { "origin": [2, 6, 0], "size": [1, 1, 1], "uv": [0, 41] }
                  ]
                }
              ]
            }
          ]
        }
  1. ใส่รูปภาพ (Texture): นำไฟล์ Texture ของปลาในฟองน้ำไปใส่ไว้ในโฟลเดอร์
    custom_block_resource_pack/
    textures/blocks
  2. ตั้งค่า Texture Map: ไปที่ custom_block_resource_pack/
    /textures     แล้วแก้ไขไฟล์ terrain_texture.json เพื่อเพิ่มชื่ออ้างอิง
  3. ตั้งชื่อบล็อก (Lang): ไปที่ custom_block_resource_pack/
    /texts/en_US.lang     แล้วเพิ่มชื่อภาษาอังกฤษให้บล็อกใหม่ของเรา
  4. เพิ่มเสียง (Optional): คุณสามารถเพิ่มข้อมูลใน blocks.json เพื่อใส่เสียงได้ (แต่เนื่องจากเราใช้ Custom Geometry ข้อมูลอื่นๆ ในไฟล์นี้จะไม่ถูกนำไปใช้ ดังนั้นขั้นตอนนี้ ไม่บังคับ ครับ สำหรับบล็อกนี้เราจะข้ามไป)

การตั้งค่า Behavior Pack

สร้างไฟล์ JSON สำหรับบล็อก

ได้เวลาสร้างไฟล์ Behavior Pack กันแล้วครับ ให้สร้างไฟล์ bubble_fish.json ในโฟลเดอร์ blocks และใส่โค้ดพื้นฐานดังนี้: 

{
          "format_version": "1.19.40",
          "minecraft:block": {
            "description": {
              "identifier": "demo:bubble_fish"
            },
            "components": {
              "minecraft:geometry": "geometry.bubble_fish",
              "minecraft:material_instances": {
                "*": {
                  "texture": "bubble_fish"
                }
              }
            }
          }
        }

เช่นเคยครับ เรากำหนด Identifier, อ้างอิง Geometry, และระบุ Texture ในกรณีนี้เราจะใช้ bubble_fish กับทุกด้าน (*) เลยครับ

วิธีการแสดงผล Render Methods

มาถึงของดีกันแล้ว! เราจะมาสำรวจพารามิเตอร์ render_method ที่อยู่ใน material_instances กันครับ ค่าเริ่มต้นของมันคือ Opaque (ทึบแสง) มาดูกันว่าถ้าเราเปลี่ยนค่าเป็นแบบต่างๆ ผลลัพธ์ที่ได้กับเจ้าปลาในฟองอากาศจะเป็นอย่างไร

1. แบบทึบแสง Opaque

นี่คือค่าพื้นฐานครับ 

  "minecraft:material_instances": {
             "*": {
              "texture": "bubble_fish",
              "render_method": "opaque"
             }
          }

ผลลัพธ์: อย่างที่คุณเห็น ความโปร่งใส (Transparency) และความโปร่งแสง (Translucency) จะไม่แสดงผลเลย สีเหล่านั้นจะถูกแสดงเป็น "สีทึบที่ใกล้เคียงที่สุด" แทน (กลายเป็นก้อนสีดำๆ ทึบๆ)

2. แบบสองด้าน Double-Sided

  "render_method": "double_sided"

ตัวเลือกนี้จะปิดการทำงานของ Backface Culling

เกร็ดความรู้: 'Backface culling' คือเทคนิคการเรนเดอร์เพื่อประหยัดทรัพยากรเครื่อง โดยเกมจะไม่แสดงผลผิวของบล็อกด้านที่คุณมองไม่เห็น (เช่น ผิวด้านในของบล็อกทึบ)

ปกติแล้ว opaque จะเปิดใช้ Backface culling เป็นค่าเริ่มต้น แต่ถ้าคุณต้องการให้แสดงผลผิวด้านหลังด้วย (หรือทำให้บล็อกมี "สองด้าน") ให้ใช้ double_sided ครับ

ผลลัพธ์: ถ้ามองจากภายนอก opaque กับ double_sided จะดูเหมือนกันเป๊ะ แต่ความต่างคือถ้าคุณ "เอาหัวมุดเข้าไปในบล็อก" แบบ Opaque คุณจะมองไม่เห็นผิวด้านใน แต่แบบ Double-Sided คุณจะมองเห็นผนังด้านในครับ

3. แบบ Alpha Test (alpha_test)

  "render_method": "alpha_test"

ผลลัพธ์: แบบนี้จะไม่รองรับความ "กึ่งใส" (Translucency) ทำให้พิกเซลที่เป็นสีฟ้าจางๆ ของเราหายไป แต่จะรองรับความ "โปร่งใส" (Transparent) และ "ทึบ" (Opaque) นอกจากนี้ยัง ปิด Backface culling ด้วย ทำให้เราสามารถมองทะลุส่วนที่ใสเข้าไปเห็นผนังด้านหลังของฟองอากาศได้... อย่างน้อยตอนนี้เราก็เห็นตัวปลาแล้ว!

4. แบบ Alpha Test หน้าเดียว (alpha_test_single_sided)

  "render_method": "alpha_test_single_sided"

วิธีนี้คล้ายกับ Alpha Test ปกติ คือไม่รองรับความกึ่งใส (Translucency) แต่รองรับความใส (Transparent) สิ่งที่ต่างกันคือ มันเปิดใช้ Backface culling

ผลลัพธ์: คุณจะมองทะลุได้ แต่จะไม่เห็นผนังด้านหลังของฟองอากาศ เหมือนกับบล็อกกระจกปกติในเกมนั่นเองครับ

5. แบบผสมผสาน Blend

  "render_method": "blend"

ผลลัพธ์: Blend อนุญาตให้แสดงผลได้ครบทั้ง 3 แบบ คือ โปร่งใส (Transparent), โปร่งแสง/กึ่งใส (Translucent), และทึบแสง (Opaque) และมันเปิดใช้ Backface culling เป็นค่าเริ่มต้น (ทำให้ไม่เห็นผนังด้านหลังที่รกตา)

ตัวเลือกแสงเงา Lighting Options

นอกจากเรื่องการแสดงผลพื้นผิวแล้ว ยังมีตัวเลือกเกี่ยวกับแสงเงาอีก 2 อย่างที่น่าสนใจให้เราสำรวจกันครับ

1. การบดบังแสงโดยรอบ Ambient Occlusion

ค่านี้กำหนดว่าควรจะเกิดเงาขึ้นรอบๆ หรือใต้บล็อกหรือไม่
ถ้าตั้งค่าเป็น true (ค่าเริ่มต้น) เกมจะสร้างเงาตรงมุมหรือซอกหลืบของบล็อก ทำให้ดูมีมิติ 

"minecraft:material_instances": {
             "*": {
              "texture": "bubble_fish",
              "render_method": "blend",
              "ambient_occlusion": true
             }
        }

จากภาพตัวอย่าง บล็อกทาง ซ้าย ตั้งค่า ambient_occlusion เป็น true (สังเกตว่าจะมีเงาที่ฐานบล็อก) ส่วนบล็อกทาง ขวา ตั้งค่าเป็น false (บล็อกดูสว่างเท่ากันหมดและไม่มีเงาที่ฐาน)

2. การหรี่แสงตามด้าน Face Dimming

ค่านี้กำหนดว่า "พื้นผิวควรมืดลงตามด้านที่มันหันไปหาแสงหรือไม่?"
ปกติแล้วใน Minecraft ด้านบนจะสว่างที่สุด ด้านข้างจะมืดลงนิดหน่อย และด้านล่างจะมืดสุด เพื่อให้วัตถุดูเป็น 3 มิติ 

"minecraft:material_instances": {
             "*": {
              "texture": "bubble_fish",
              "render_method": "blend",
              "face_dimming": true
             }
        }

เช่นเดียวกับตัวอย่างก่อนหน้าครับ บล็อกทาง ซ้าย ตั้งค่า face_dimming เป็น true (แสงแต่ละด้านไม่เท่ากัน ดูมีมิติ) ส่วนบล็อกทาง ขวา ตั้งค่าเป็น false (แสงสว่างเท่ากันทุกด้าน ทำให้ดู "แบน" หรือเหมือนเรืองแสงได้นิดหน่อย)
