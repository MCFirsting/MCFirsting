การสร้างไอเท็มแบบกำหนดเอง Custom Items

เปลี่ยนไอเดียให้เป็นไอเท็มในมือคุณ

เช่นเดียวกับที่คุณสามารถสร้างบล็อกและสิ่งมีชีวิต (Entities) ขึ้นมาเองได้ คุณก็สามารถสร้าง ไอเท็ม (Items) แบบกำหนดเองใน Minecraft ได้เช่นกันครับ!

ในบทช่วยสอนนี้ เราจะมาดูวิธีการทำกันครับ ว่าไอเท็มแต่ละประเภทต้องใช้ไฟล์แบบไหน ต้องนำไฟล์เหล่านั้นไปวางไว้ตรงไหนใน Resource Pack และ Behavior Pack รวมถึงต้องสร้างหรือแก้ไขไฟล์ JSON ตัวไหนบ้าง

⚠️ ข้อควรระวัง

เช่นเดียวกับ Entity และ Block ครับ ไฟล์สำหรับไอเท็มนั้นจะมี ชื่อเฉพาะ (หรือนามสกุลไฟล์) ที่กำหนดไว้ และจำเป็นต้องถูกวางไว้ใน โฟลเดอร์หรือโฟลเดอร์ย่อยที่ถูกต้อง อย่างเคร่งครัด

หาก Add-ons ไอเท็มใหม่ของคุณใช้งานไม่ได้ สิ่งแรกที่ควรทำคือ ตรวจสอบอีกครั้งว่าวางไฟล์ทุกอย่างถูกที่หรือไม่!

สิ่งที่ควรรู้ก่อนเริ่ม

คุณควรทำความคุ้นเคยกับบทเรียนเหล่านี้มาก่อนที่จะเริ่มทำ Custom Items นะครับ:

  • การเริ่มต้นพัฒนา Add-ons
  • ความรู้เบื้องต้นเรื่อง Resource Pack
  • ความรู้เบื้องต้นเรื่อง Behavior Pack
  • การสร้าง Custom Block

นอกจากนี้ คุณอาจจะลองเข้าไปดูโครงสร้างไฟล์และโฟลเดอร์ของ Vanilla Minecraft Add-On Packs (แพ็กต้นฉบับของเกม) เพื่อดูตัวอย่างประกอบความเข้าใจได้อีกด้วยครับ

ลงมือสร้างไอเท็มกันเถอะ!

เราจะมาลองสร้างไอเท็ม 4 อย่างที่ไม่มีอยู่ใน Minecraft ปกติ (Vanilla) กันครับ โดยจะเรียงลำดับตามความซับซ้อนดังนี้:

  1. Goo (น้ำเมือก)
  2. Wrench (ประแจ)
  3. Crown (มงกุฎ)
  4. Custom Chestplate with Vanilla Armor Trim (เกราะอกแบบสร้างเองที่รองรับระบบลวดลายชุดเกราะ)

และในตอนท้าย เราจะพูดถึงการทำไอเท็มแบบ ย้อมสีได้ (Dyeable custom items) กันด้วยครับ

คุณสามารถเลือกสร้าง Add-ons Packs แยกสำหรับแต่ละไอเท็ม หรือจะรวมหลายไอเท็มไว้ในแพ็กชุดเดียวกันก็ได้ครับ

กระบวนการทำงานทั่วไป

นี่คือขั้นตอนมาตรฐานที่เราจะใช้ในการสร้างไอเท็มแต่ละชิ้นครับ:

  1. สร้างโฟลเดอร์ Behavior Pack และ Resource Pack
  2. สร้างโครงสร้างโฟลเดอร์และไฟล์สำหรับไอเท็มที่ต้องการ รวมถึงไฟล์ manifest.json
  3. คัดลอก (Copy) และวาง (Paste) เนื้อหาโค้ดของไอเท็มจากบทเรียนลงไปในไฟล์
  4. ดาวน์โหลดและบันทึกไฟล์รูปภาพลงในตำแหน่งที่ถูกต้อง
  5. คัดลอกหรือย้ายแพ็กที่เสร็จแล้วไปไว้ในโฟลเดอร์ development_behavior_pack และ development_resource_pack ในโฟลเดอร์ com.mojang ของเครื่องคุณ
  6. เปิดเกม Minecraft: Bedrock Edition
  7. สร้างโลกใหม่ (Create World) โดยเปิดใช้งาน Behavior Pack และ Resource Pack ที่เราสร้าง
  8. เลือกโหมด Creative หรือ Survival (ที่เปิด Cheats ไว้)
  9. เข้าสู่โลกเกมและใช้คำสั่ง /give @s demo:<ชื่อไอเท็ม> เพื่อเสกไอเท็มของคุณออกมา

เคล็ดลับ

แม้ว่าในบทเรียนนี้เราจะเตรียมไฟล์รูปภาพและโมเดลไว้ให้แล้ว แต่คุณสามารถแก้ไขหรือสร้างใหม่ทั้งหมดได้ด้วยโปรแกรม Blockbench หรือโปรแกรมแก้ไขอื่นๆ ได้ตามใจชอบเลยครับ

ไฟล์ Manifest

ไฟล์ manifest.json ของแต่ละโปรเจกต์จะใช้แม่แบบที่คล้ายคลึงกัน (เหมือนกับในบทเรียน Resource Pack และ Behavior Pack ก่อนหน้านี้) แต่อย่าลืมสร้าง UUID ใหม่สำหรับโปรเจกต์ของคุณเองด้วยนะครับ!

ตัวอย่าง Behavior Pack manifest.json:

{
          "format_version": 2,
          "header": {
            "description": "Custom Item Behavior Pack",
            "name": "Custom Item Behavior Pack",
            "uuid": "<ADD YOUR OWN UUID>",
            "version": [1, 0, 1],
            "min_engine_version": [1, 21, 80]
          },
          "modules": [
            {
              "description": "Custom Item Behavior Pack",
              "type": "data",
              "uuid": "<ADD YOUR OWN UUID>",
              "version": [1, 0, 1]
            }
          ]
        }

ตัวอย่าง Resource Pack manifest.json:

{
          "format_version": 2,
          "header": {
            "description": "Custom Item Resource Pack",
            "name": "Custom Item Resource Pack",
            "uuid": "<ADD YOUR OWN UUID>",
            "version": [1, 0, 1],
            "min_engine_version": [1, 21, 80]
          },
          "modules": [
            {
              "description": "Custom Item Resource Pack",
              "type": "resources",
              "uuid": "<ADD YOUR OWN UUID>",
              "version": [1, 0, 1]
            }
          ]
        }

1. Goo (น้ำเมือก)

Goo เป็นไอเท็มแบบกำหนดเองที่ง่ายที่สุดครับ คือคุณไม่ได้สวมใส่หรือกินมัน มันก็แค่กำมือของน้ำเมือกแปลกๆ เท่านั้นเอง แต่มันมีประโยชน์มากสำหรับตรวจสอบว่า Add-ons Packs ของคุณทำงานได้ปกติหรือไม่ และเอาไว้ทดลองเรื่องขนาดกอง (Stack size) และชื่อที่แสดงในเกม (Display name) ครับ

การตั้งค่า Behavior Pack

ให้สร้างไฟล์ชื่อ goo.json ในโฟลเดอร์ development_behavior_pack/items/ แล้วคัดลอก JSON ด้านล่างนี้ไปวางครับ: 

{
          "format_version": "1.21.80",
          "minecraft:item": {
            "description": {
              "identifier": "demo:goo",
              "category": "Items"
            },
            "components": {
              "minecraft:max_stack_size": 16,
              "minecraft:icon": {
                "texture": "demo:goo"
              },
              "minecraft:display_name": {
                "value": "Weird Goo"
              }
            }
          }
        }

การตั้งค่า Resource Pack

ในฝั่ง Resource Pack เราต้องกำหนดว่าไอเท็ม demo:goo จะใช้รูปภาพไหน โดยการแก้ไขไฟล์ item_texture.json ในโฟลเดอร์ development_resource_pack/textures/ ดังนี้ครับ: 

{
          "resource_pack_name": "Goo Resource Pack",
          "texture_data": {
            "demo:goo": {
              "textures": "textures/items/goo"
            }
          }
        }

รูปภาพ Texture

นี่คือไฟล์ PNG สำหรับใช้เป็น Texture ของ Goo ครับ:

ให้คัดลอกไฟล์ภาพที่ดาวน์โหลดมาไปวางไว้ที่โฟลเดอร์ development_resource_pack/textures/items/ นะครับ

ผลลัพธ์

หลังจากที่คุณสร้าง Packs ทั้งสองส่วนเสร็จแล้ว ให้เข้าไปในเกม Minecraft และลองเสก Goo ให้ตัวเอง 

/give @s demo:goo

2. Wrench (ประแจยักษ์)

ประแจยักษ์สุดเท่! ความพิเศษของไอเท็มนี้คือมันมีไฟล์ Texture แยกกันครับ คือไฟล์สำหรับแสดงในช่องเก็บของ (Hotbar) แบบ 2D และไฟล์สำหรับแสดงเป็นโมเดล 3D เวลาที่คุณถือมันอยู่ในมือ

การตั้งค่า Behavior Pack

ให้สร้างไฟล์ wrench.json ในโฟลเดอร์ development_behavior_pack/items/ แล้ววางโค้ดนี้ลงไปครับ: 

{
          "format_version": "1.21.80",
          "minecraft:item": {
            "description": {
              "identifier": "demo:wrench",
              "category": "Items"
            },
            "components": {
              "minecraft:max_stack_size": 16,
              "minecraft:icon": {
                "texture": "demo:wrench"
              },
              "minecraft:wearable": {
                "dispensable": true,
                "slot": "slot.weapon.offhand"
              }
            }
          }
        }

การตั้งค่า Resource Pack

สำหรับ Wrench เราต้องสร้างไฟล์หลายส่วนใน Resource Pack เพื่อประกอบร่างมันเข้าด้วยกันครับ

1. ไฟล์ Attachable (กำหนดการยึดติดกับตัวละคร)

สร้างไฟล์ wrench.player.json ในโฟลเดอร์ development_resource_pack/attachables/ (ถ้าไม่มีโฟลเดอร์ attachables ให้สร้างขึ้นมาใหม่ครับ): 

{
          "format_version": "1.20.30",
          "minecraft:attachable": {
            "description": {
              "identifier": "demo:wrench",
              "item": { "demo:wrench": "query.is_owner_identifier_any('minecraft:player')" },
              "materials": {
                "default": "entity",
                "enchanted": "entity_alphatest_glint"
              },
              "textures": {
                "default": "textures/items/wrench",
                "enchanted": "textures/misc/enchanted_item_glint"
              },
              "geometry": {
                "default": "geometry.wrench"
              },
              "animations": {
                "hold_first_person": "animation.steve_head.hold_first_person",
                "hold_third_person": "animation.steve_head.hold_third_person"
              },
              "scripts": {
                "animate": [
                  {
                    "hold_first_person": "context.is_first_person == 1.0"
                  },
                  {
                    "hold_third_person": "context.is_first_person == 0.0"
                  }
                ]
              },
              "render_controllers": ["controller.render.item_default"]
            }
          }
        }

3. ไฟล์รูปภาพ

เราต้องการไฟล์ 2 รูปครับ ให้ดาวน์โหลดแล้วนำไปวางที่โฟลเดอร์ development_resource_pack/textures/items

1. ไอคอน (wrench_ico.png): สำหรับแสดงในช่องเก็บของ

2. สกินโมเดล (wrench.png): สำหรับแสดงบนโมเดล 3D

4. กำหนด Texture ไอเท็ม

สุดท้าย ให้กลับไปแก้ไขไฟล์ item_texture.json ในโฟลเดอร์ textures เพื่อเพิ่มการอ้างอิงของ wrench เข้าไปครับ: 

{
          "resource_pack_name": "Wrench Resource Pack",
          "texture_data": {
            "demo:wrench": {
              "textures": "textures/items/wrench_ico"
            }
          }
        }

(สังเกตว่าในไฟล์นี้เราลิงก์ไปที่ wrench_ico นะครับ เพราะเราต้องการให้ไอคอนในช่องเก็บของเป็นรูป 2D ส่วนโมเดล 3D มันจะไปดึงรูป wrench จากไฟล์ attachables เองครับ)

3. Crown (มงกุฎ)

เราจะมาสร้างมงกุฎราชากันครับ โดยใช้คุณสมบัติการสวมใส่แบบหมวก (Helmet) แต่เปลี่ยนรูปร่างหน้าตาใหม่หมด

การตั้งค่า Behavior Pack

สร้างไฟล์ crown.json ในโฟลเดอร์ development_behavior_pack/items/ และวางโค้ดนี้ครับ: 

{
          "format_version": "1.21.80",
          "minecraft:item": {
            "description": {
              "identifier": "demo:crown",
              "category": "Items"
            },
            "components": {
              "minecraft:max_stack_size": 1,
              "minecraft:icon": {
                "texture": "demo:crown"
              },
              "minecraft:wearable": {
                "dispensable": true,
                "slot": "slot.armor.head"
              }
            }
          }
        }

การตั้งค่า Resource Pack

ในส่วนนี้เราต้องจัดการไฟล์ 3 ส่วนหลักๆ คือ Attachable, Geometry (โมเดล), และ Texture ครับ

1. ไฟล์ Attachable (crown.player.json)

สร้างไฟล์ crown.player.json ในโฟลเดอร์ development_resource_pack/attachables/:

(สังเกตส่วน scripts นะครับ มีการสั่งปิดเลเยอร์หมวกเดิม variable.helmet_layer_visible = 0.0; เพื่อไม่ให้ภาพซ้อนกัน)

{
          "format_version": "1.19.80",
          "minecraft:attachable": {
            "description": {
              "identifier": "demo:crown_hat.player",
              "item": { "demo:crown": "query.is_owner_identifier_any('minecraft:player')" },
              "materials": {
                "default": "armor",
                "enchanted": "armor_enchanted"
              },
              "textures": {
                "default": "textures/items/crown_minecraft",
                "enchanted": "textures/misc/enchanted_item_glint"
              },
              "geometry": {
                "default": "geometry.crown"
              },
              "scripts": {
                "parent_setup": "variable.helmet_layer_visible = 0.0;"
              },
              "render_controllers": ["controller.render.armor"]
            }
          }
        }

5. ไอเท็มแบบย้อมสีได้ (Dyeable custom items)

ตอนนี้ไอเท็มแบบกำหนดเองสามารถนำไปย้อมสีในหม้อปรุงยา (Cauldrons) ได้แล้วครับ! แต่มีข้อกำหนดว่า format_version ของไฟล์ไอเท็มและ attachable จะต้องเป็นเวอร์ชัน 1.21.30 หรือสูงกว่าเท่านั้น

ในหัวข้อนี้ เราจะสมมติว่าคุณทำ "เกราะอกแบบกำหนดเอง" จากหัวข้อที่แล้วเสร็จเรียบร้อยแล้วนะครับ ให้ Copy Add-ons Pack ชุดนั้นมาเพื่อแก้ไขต่อได้เลย

1. แก้ไข Behavior Pack

เปิดไฟล์ custom_chestplate.item.json ใน Behavior Pack ขึ้นมาแก้ไขดังนี้ครับ:

  1. เปลี่ยน format_version ให้เป็น "1.21.30"
  2. แก้ไขส่วน minecraft:icon ให้รองรับทั้งแบบปกติ (default) และแบบย้อมสี (dyed)
  3. เพิ่ม component minecraft:dyeable เข้าไป (กำหนดสีเริ่มต้นด้วย Hex code เช่น #175882)

ไฟล์ JSON ฉบับสมบูรณ์จะมีหน้าตาแบบนี้ครับ:

{ 
          "format_version": "1.21.30", 
          "minecraft:item": { 
            "description": { 
              "identifier": "demo:custom_chestplate", 
              "menu_category": { 
                "category": "equipment", 
                "group": "itemGroup.name.chestplate" 
              }
            },
            "components": { 
              "minecraft:max_stack_size": 1, 
              "minecraft:icon": {
                "textures": {
                  "default": "custom_chestplate_icon",
                  "dyed": "custom_chestplate_icon_gray"
                }
              },
              "minecraft:wearable": {
                "protection": 10, 
                "slot": "slot.armor.chest" 
              }, 
              "minecraft:durability": { 
                "damage_chance": { 
                  "min": 10, 
                  "max": 50 
              }, 
                "max_durability": 1560 
              }, 
              "minecraft:repairable": { 
                "repair_items": [ 
                  { 
                    "items": ["pink_dye"], 
                    "repair_amount": 390 
                  } 
                ] 
              }, 
              "minecraft:tags": { 
                "tags": [ 
                  "minecraft:is_armor", 
                  "minecraft:trimmable_armors" 
                ] 
              },
              "minecraft:dyeable": {
                "default_color":  "#175882"
              }
            } 
          } 
        }

(ถ้าคุณไม่ต้องการให้มีสีเริ่มต้น ก็สามารถลบส่วน default_color ออกได้ครับ ไอเท็มจะแสดงสีปกติจนกว่าจะถูกย้อม)

2. เตรียมไฟล์รูปภาพ TGA

เพื่อให้ไอคอนสามารถเปลี่ยนสีได้ถูกต้อง เราจำเป็นต้องใช้ไฟล์ไอคอนเวอร์ชัน ขาว-ดำ และต้องเป็นนามสกุล .tga ครับ

วิธีทำ: ให้เอารูปไอคอนเดิมมาแปลงเป็นขาวดำในโปรแกรมแต่งภาพ แล้วบันทึกไฟล์เป็นนามสกุล .tga (ตั้งชื่อไฟล์ว่า custom_chestplate_icon_grayscale.tga) แล้วนำไปใส่ในโฟลเดอร์ Texture ของคุณครับ

3. แก้ไข Resource Pack - Attachable

แก้ไขไฟล์ custom_chestplate.attachable.json โดยเปลี่ยน version เป็น 1.21.30 และปรับปรุงส่วนต่างๆ ดังนี้:

  • render_controllers: เปลี่ยนให้ชี้ไปที่ controller ตัวใหม่ที่เราจะสร้าง
  • materials: เพิ่ม material สำหรับตอนย้อมสี (dyed)
  • textures: เพิ่ม texture ไฟล์ .tga สำหรับตอนย้อมสี
  • scripts: เพิ่มตัวแปรเพื่อเช็คว่าเกราะถูกย้อมสีอยู่หรือไม่
{
          "format_version": "1.21.30", 
          "minecraft:attachable": { 
            "description": { 
              "identifier": "demo:custom_chestplate", 
              "render_controllers": ["controller.render.custom_chestplate"], 
              "materials": { 
                "default": "armor", 
                "dyed": "entity_alphatest_change_color"
              }, 
              "textures": { 
                "default": "textures/items/custom_chestplate.png",
                "dyed": "textures/items/custom_chestplate_grayscale.tga"
              }, 
              "geometry": { 
                "default": "geometry.custom_chestplate"
              },
              "scripts": {
                "pre_animation": [
                  "variable.is_dyed = query.armor_color_slot(1, 0) != 1.0 || query.armor_color_slot(1, 1) != 1.0 || query.armor_color_slot(1, 2) != 1.0"
                ]
              }
            }
          }
        }

4. สร้าง Render Controller

ใน Resource Pack ให้สร้างโฟลเดอร์ชื่อ render_controllers และสร้างไฟล์ใหม่ชื่อ custom_chestplate.render_controllers.json ใส่โค้ดนี้ครับ:

(ไฟล์นี้ทำหน้าที่สลับ Material และ Texture ระหว่างแบบปกติและแบบย้อมสี ตามตัวแปร variable.is_dyed ที่เราเขียนสคริปต์ไว้)

{
          "format_version": "1.8.0",
          "render_controllers": {
            "controller.render.custom_chestplate": {
              "geometry": "Geometry.default",
              "materials": [ { "*": "variable.is_dyed ? Material.dyed : Material.default" }
        ],
              "textures": [
                "variable.is_dyed ? Texture.dyed : Texture.default"
              ]
            }
          }
        }

5. อัปเดต Item Texture Definition

สุดท้าย แก้ไขไฟล์ development_resource_pack/textures/item_texture.json ให้รู้จัก Texture ทั้งสองแบบครับ: 

{
          "texture_data": {
            "custom_chestplate": {
              "textures": "textures/items/custom_chestplate.png"
            },
            "custom_chestplate_icon_gray": {
              "textures": "textures/items/custom_chestplate_icon_grayscale.tga"
            }
          }
        }
